### **Тема**
Настройка много-зонной архитектуры OSPF с различными типами зон: магистральная, тупиковая, totally stub и зона для распространения внешних маршрутов.

### **Цель работы**
- **Теоретическая:** Изучить принципы организации OSPF с несколькими зонами, типы зон OSPF (backbone, stub, totally stub, NSSA) и их особенности.
- **Практическая:** Приобрести навыки настройки много-зонного OSPF на оборудовании Cisco, управления распространением маршрутов между зонами, настройки суммаризации и распространения внешних маршрутов.

### **Задачи**
1.  Построить сетевую топологию с 6 маршрутизаторами, разделенными на 4 зоны OSPF.
2.  Настроить IP-адресацию на всех интерфейсах.
3.  Настроить OSPF с указанием принадлежности интерфейсов к соответствующим зонам.
4.  Настроить зону 1 как тупиковую (stub).
5.  Настроить зону 2 как totally stub.
6.  Настроить зону 3 как NSSA (Not-So-Stubby Area) для импорта внешнего статического маршрута.
7.  Настроить статический маршрут до внешней сети на пограничном маршрутизаторе и распространить его в OSPF.
8.  Проверить связность между всеми сетями и проанализировать таблицы маршрутизации в разных зонах.

### **Оборудование и программное обеспечение**
- **Программное обеспечение:** Cisco Packet Tracer
- **Оборудование:**
  - Маршрутизаторы Cisco 1941: 6 шт. (R1, R2, R3, R4, R5, R6)
  - Коммутаторы Cisco 2960: 3 шт. (SW1, SW2, SW3)
  - Персональные компьютеры: 3 шт. (PC1, PC2, PC3)
  - Соединительные провода: медные прямые (straight-through) и последовательные (serial)

### **Краткие теоретические сведения**

**Много-зонная архитектура OSPF** используется для уменьшения размера баз данных состояний каналов (LSDB) и уменьшения служебного трафика в больших сетях. Основные типы зон:

1. **Backbone Area (Area 0):** Магистральная зона, обязательная для всех конфигураций OSPF с несколькими зонами. Все остальные зоны должны быть подключены к Area 0.

2. **Stub Area:** Тупиковая зона, в которую не поступают внешние маршруты (Type 5 LSA). Вместо них ABR (Area Border Router) инжектирует маршрут по умолчанию (0.0.0.0/0).

3. **Totally Stub Area:** Полностью тупиковая зона, в которую не поступают ни внешние маршруты (Type 5 LSA), ни маршруты из других зон (Type 3 LSA). Только маршрут по умолчанию.

4. **NSSA (Not-So-Stubby Area):** "Не совсем тупиковая" зона, позволяющая импортировать внешние маршруты (Type 7 LSA) из локальной AS, которые затем преобразуются в Type 5 LSA на ABR.

**Роли маршрутизаторов в много-зонном OSPF:**
- **Internal Router:** Все интерфейсы в одной зоне.
- **Area Border Router (ABR):** Интерфейсы в нескольких зонах.
- **Autonomous System Boundary Router (ASBR):** Распространяет внешние маршруты в OSPF.

**Типы LSA в много-зонной архитектуре:**
- **Type 1 (Router LSA):** В пределах одной зоны.
- **Type 2 (Network LSA):** В пределах одной зоны.
- **Type 3 (Summary LSA):** Распространяются между зонами через ABR.
- **Type 4 (ASBR Summary LSA):** Информация о местоположении ASBR.
- **Type 5 (External LSA):** Внешние маршруты от ASBR.
- **Type 7 (NSSA External LSA):** Внешние маршруты в NSSA.

### **Порядок выполнения работы**

#### **1. Подготовительный этап**
1.  Запустите Cisco Packet Tracer.
2.  Создайте новую рабочую область.
3.  Добавьте оборудование согласно следующей схеме приложенной к работе:

#### **2. Основной этап**

1.  **Настройка IP-адресации на маршрутизаторах:**

    **R1:**
    ```
    interface Loopback0
      ip address 1.1.1.1 255.255.255.255
    interface Serial0/0/0
      ip address 10.0.12.1 255.255.255.252
      no shutdown
    interface Serial0/0/1
      ip address 10.0.13.1 255.255.255.252
      no shutdown
    interface FastEthernet0/0
      ip address 10.1.16.1 255.255.255.0
      no shutdown
    ```

    **R2 (ABR для Area 0 и 1):**
    ```
    interface Loopback0
      ip address 2.2.2.2 255.255.255.255
    interface Serial0/0/0
      ip address 10.0.12.2 255.255.255.252
      no shutdown
    interface FastEthernet0/0
      ip address 172.16.24.2 255.255.255.0
      no shutdown
    ```

    **R3 (ABR для Area 0 и 2):**
    ```
    interface Loopback0
      ip address 3.3.3.3 255.255.255.255
    interface Serial0/0/0
      ip address 10.0.13.2 255.255.255.252
      no shutdown
    interface FastEthernet0/0
      ip address 192.168.35.3 255.255.255.0
      no shutdown
    ```

    **R4 (Internal router Area 1):**
    ```
    interface Loopback0
      ip address 4.4.4.4 255.255.255.255
    interface FastEthernet0/0
      ip address 172.16.24.4 255.255.255.0
      no shutdown
    interface FastEthernet0/1
      ip address 172.16.1.1 255.255.255.0
      no shutdown
    ```

    **R5 (Internal router Area 2):**
    ```
    interface Loopback0
      ip address 5.5.5.5 255.255.255.255
    interface FastEthernet0/0
      ip address 192.168.35.5 255.255.255.0
      no shutdown
    interface FastEthernet0/1
      ip address 192.168.5.1 255.255.255.0
      no shutdown
    ```

    **R6 (ASBR/NSSA Area 3):**
    ```
    interface Loopback0
      ip address 6.6.6.6 255.255.255.255
    interface FastEthernet0/0
      ip address 10.1.16.6 255.255.255.0
      no shutdown
    interface FastEthernet0/1
      ip address 10.100.30.1 255.255.255.0
      no shutdown
    interface Loopback100  (эмулируем внешнюю сеть)
      ip address 203.0.113.1 255.255.255.0
    ```

2.  **Настройка OSPF:**

    **R1 (Internal router Area 0):**
    ```
    router ospf 1
      router-id 1.1.1.1
      network 1.1.1.1 0.0.0.0 area 0
      network 10.0.12.0 0.0.0.3 area 0
      network 10.0.13.0 0.0.0.3 area 0
      network 10.1.16.0 0.0.0.255 area 3
    ```

    **R2 (ABR Area 0/1):**
    ```
    router ospf 1
      router-id 2.2.2.2
      network 2.2.2.2 0.0.0.0 area 0
      network 10.0.12.0 0.0.0.3 area 0
      network 172.16.24.0 0.0.0.255 area 1
      area 1 stub   ! Настройка зоны 1 как stub
    ```

    **R3 (ABR Area 0/2):**
    ```
    router ospf 1
      router-id 3.3.3.3
      network 3.3.3.3 0.0.0.0 area 0
      network 10.0.13.0 0.0.0.3 area 0
      network 192.168.35.0 0.0.0.255 area 2
      area 2 stub no-summary   ! Настройка зоны 2 как totally stub
    ```

    **R4 (Internal router Area 1):**
    ```
    router ospf 1
      router-id 4.4.4.4
      network 4.4.4.4 0.0.0.0 area 1
      network 172.16.24.0 0.0.0.255 area 1
      network 172.16.1.0 0.0.0.255 area 1
      area 1 stub   ! Должно совпадать с настройкой на ABR
    ```

    **R5 (Internal router Area 2):**
    ```
    router ospf 1
      router-id 5.5.5.5
      network 5.5.5.5 0.0.0.0 area 2
      network 192.168.35.0 0.0.0.255 area 2
      network 192.168.5.0 0.0.0.255 area 2
      area 2 stub   ! Должно совпадать с настройкой на ABR
    ```

    **R6 (ASBR Area 3 - NSSA):**
    ```
    router ospf 1
      router-id 6.6.6.6
      network 6.6.6.6 0.0.0.0 area 3
      network 10.1.16.0 0.0.0.255 area 3
      network 10.100.30.0 0.0.0.255 area 3
      area 3 nssa   ! Настройка зоны 3 как NSSA
    ! Настройка статического маршрута и его распространение
    ip route 203.0.113.0 255.255.255.0 Null0
    router ospf 1
      redistribute static subnets
    ```

3.  **Проверка настройки:**

    - На R1: `show ip ospf neighbor` - должны быть соседи R2, R3, R6
    - На R2: `show ip ospf interface brief` - проверка принадлежности интерфейсов к зонам
    - На R3: `show ip ospf` - проверка конфигурации OSPF

4.  **Проверка таблиц маршрутизации:**

    **На R4 (Stub Area):**
    ```
    show ip route ospf
    ```
    Должны быть видны:
    - Маршруты к сетям Area 0 (через Type 3 LSA)
    - Маршрут по умолчанию (0.0.0.0/0)
    - НЕ должно быть маршрутов к внешней сети 203.0.113.0/24

    **На R5 (Totally Stub Area):**
    ```
    show ip route ospf
    ```
    Должен быть только:
    - Маршрут по умолчанию (0.0.0.0/0)
    - НЕ должно быть маршрутов к другим зонам и внешним сетям

    **На R1 (Backbone):**
    ```
    show ip route ospf
    ```
    Должны быть все маршруты, включая:
    - Сети Area 1 и Area 2
    - Сеть Area 3
    - Внешняя сеть 203.0.113.0/24 (как внешний маршрут E2)

5.  **Проверка базы данных OSPF:**

    **На R1:**
    ```
    show ip ospf database
    ```
    Обратите внимание на разные типы LSA:
    - Type 1: Для каждого роутера в Area 0
    - Type 3: Summary LSA для сетей из других зон
    - Type 5: External LSA для сети 203.0.113.0/24
    - Type 4: ASBR Summary LSA (если есть)

    **На R6:**
    ```
    show ip ospf database nssa-external
    ```
    Покажет Type 7 LSA, которые генерируются в NSSA.
