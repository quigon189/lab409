### **Тема**
Поиск и устранение неисправностей в работе протокола OSPFv2 в одной зоне (Area 0).

### **Цель работы**
- **Теоретическая:** Изучить типичные проблемы при работе OSPF, методы диагностики и последовательность поиска неисправностей в сети OSPF.
- **Практическая:** Приобрести навыки использования диагностических команд OSPF в FRR, анализа таблиц маршрутизации и соседства, устранения типовых ошибок конфигурации.

### **Задачи**
1.  Развернуть сетевую топологию с предустановленными ошибками конфигурации OSPF.
2.  Используя команды `ping` и `traceroute`, определить проблемные участки сети.
3.  С помощью диагностических команд OSPF выявить конкретные причины отсутствия связности.
4.  Последовательно устранить все ошибки конфигурации.
5.  Добиться полной связности между всеми подсетями.
6.  Задокументировать процесс диагностики и устранения неисправностей.

### **Оборудование и программное обеспечение**
- **Программное обеспечение:** Containerlab, Docker, FRR (`frrouting/frr:latest`), клиентские контейнеры (`alpine:latest`).
- **Оборудование (виртуальное):**
  - Маршрутизаторы FRR: 5 шт. (R1, R2, R3, R4, R5)
  - Клиентские хосты: 5 шт. (PC1, PC2, PC3, PC4, PC5)
  - Коммутаторы: 2 шт. (SW1, SW2)

### **Краткие теоретические сведения**

**Типичные проблемы в OSPF и методы их диагностики:**

1. **Отсутствие соседства (Neighbor Adjacency):**
   - Несовпадение Area ID
   - Несовпадение таймеров Hello/Dead
   - Несовпадение аутентификации
   - Неправильная маска подсети в broadcast-сетях
   - ACL блокирует multicast (224.0.0.5, 224.0.0.6)
   - Неверный MTU (особенно для виртуальных каналов)

2. **Неполная таблица маршрутизации:**
   - Отсутствие объявления сети (`network` command)
   - Пассивный (passive) интерфейс
   - Фильтрация маршрутов (distribute-list)
   - Проблемы с Router ID (дублирование)

3. **Субоптимальные маршруты:**
   - Неправильные стоимости интерфейсов
   - Проблемы с reference-bandwidth
   - Асимметричные стоимости на разных концах канала

**Ключевые диагностические команды FRR:**
- `show ip ospf neighbor` - состояние соседства
- `show ip ospf interface [brief]` - параметры OSPF на интерфейсах
- `show ip ospf database` - база данных состояний каналов
- `show ip route ospf` - маршруты, полученные через OSPF
- `show running-config` - текущая конфигурация
- `debug ospf packet hello` - отладка Hello-пакетов
- `ping` и `traceroute` - проверка связности

### **Порядок выполнения работы**

#### **1. Подготовительный этап**
1.  Создайте директорию: `mkdir ~/lab3_ospf && cd ~/lab3_ospf`
2. Создайте 2 Open vSwitch (если их нет) `SW1` и `SW2`
3.  Создайте файл `topology.yaml`:

```yaml
name: lab3-ospf

topology:
  nodes:
    SW1:
      kind: ovs-switch
    SW2:
      kind: ovs-switch

    R1:
      kind: linux
      image: frrouting/frr:latest
      binds:
        - ./configs/R1/:/etc/frr/
    R2:
      kind: linux
      image: frrouting/frr:latest
      binds:
        - ./configs/R2/:/etc/frr/
    R3:
      kind: linux
      image: frrouting/frr:latest
      binds:
        - ./configs/R3/:/etc/frr/
    R4:
      kind: linux
      image: frrouting/frr:latest
      binds:
        - ./configs/R4/:/etc/frr/
    R5:
      kind: linux
      image: frrouting/frr:latest
      binds:
        - ./configs/R5/:/etc/frr/

    PC1:
      kind: linux
      image: alpine:latest
      exec:
        - ip addr add 172.16.1.100/24 dev eth1
        - ip route add default via 172.16.1.1
    PC2:
      kind: linux
      image: alpine:latest
      exec:
        - ip addr add 172.16.2.100/24 dev eth1
        - ip route add default via 172.16.2.1
    PC3:
      kind: linux
      image: alpine:latest
      exec:
        - ip addr add 172.16.3.100/24 dev eth1
        - ip route add default via 172.16.3.1
    PC4:
      kind: linux
      image: alpine:latest
      exec:
        - ip addr add 172.16.4.100/24 dev eth1
        - ip route add default via 172.16.4.1
    PC5:
      kind: linux
      image: alpine:latest
      exec:
        - ip addr add 172.16.5.100/24 dev eth1
        - ip route add default via 172.16.5.1

  links:
    # Broadcast сегмент 1: R1, R2, R3 подключены к SW1
    - endpoints: ["SW1:eth1", "R1:eth1"]
    - endpoints: ["SW1:eth2", "R2:eth1"]
    - endpoints: ["SW1:eth3", "R3:eth1"]
    
    # Broadcast сегмент 2: R4, R5 подключены к SW2
    - endpoints: ["SW2:eth1", "R4:eth1"]
    - endpoints: ["SW2:eth2", "R5:eth1"]
    
    # Point-to-Point соединения
    - endpoints: ["R3:eth2", "R4:eth2"]  # P2P link
    - endpoints: ["R2:eth2", "R5:eth2"]  # P2P link
    
    # Клиентские сети
    - endpoints: ["R1:eth2", "PC1:eth1"]
    - endpoints: ["R2:eth3", "PC2:eth1"]
    - endpoints: ["R3:eth3", "PC3:eth1"]
    - endpoints: ["R4:eth3", "PC4:eth1"]
    - endpoints: ["R5:eth3", "PC5:eth1"]
```

3.  Создайте начальные конфигурации с ошибками:

#### **2. Основной этап**

1.  **Развертывание стенда с ошибками:**
    ```
    sudo clab deploy -t topology.yaml
    ```

2.  **Первичная диагностика связности:**
    - С PC1 выполните: `ping 172.16.2.100` (должно работать, т.к. в одной broadcast-сети)
    - С PC1 выполните: `ping 172.16.3.100` (должно работать)
    - С PC1 выполните: `ping 172.16.4.100` (не должно работать)
    - С PC1 выполните: `ping 172.16.5.100` (не должно работать)
    - С PC4 выполните: `traceroute 172.16.5.100` для анализа пути

3.  **Систематический поиск ошибок:**
    
    **Шаг 1: Проверка соседства OSPF**
    - На R1: `show ip ospf neighbor`
    - На R4: `show ip ospf neighbor`
    - Какие соседи отсутствуют? Почему?

    **Шаг 2: Проверка конфигурации интерфейсов**
    - На R1: `show ip ospf interface eth1`
    - На R2: `show ip interface brief`
    - На R3: `show ip ospf interface eth1`
    - На R4: `show running-config`

    **Шаг 3: Проверка таблицы маршрутизации**
    - На R1: `show ip route ospf`
    - На R5: `show ip route`
    - Каких маршрутов не хватает?

    **Шаг 4: Проверка базы данных OSPF**
    - На всех роутерах: `show ip ospf database`
    - Сравните выводы - есть ли различия?

4.  **Устранение ошибок (последовательность):**
    - Найдите и исправьте 15 ошибок в конфигурации маршрутизаторов

5.  **Проверка после исправлений:**
    - Перезапустите OSPF на роутерах: `clear ip ospf process`
    - Проверьте соседство: все должны видеть соседей в состоянии FULL
    - Проверьте таблицы маршрутизации: все сети должны быть видны
    - Проверьте связность: ping между всеми PC должен проходить
