### **Тема**
Построение корпоративной сети, объединяющей три географически разделенных филиала с использованием инфраструктуры провайдера связи (WAN).

### **Цель работы**
- **Теоретическая:** Изучить базовые принципы построения глобальных сетей (WAN), роль оборудования на границе с провайдером (CE – Customer Edge), технологии инкапсуляции канального уровня (HDLC, PPP) и способы организации связности между филиалами.
- **Практическая:** Приобрести навыки настройки последовательных интерфейсов, инкапсуляции PPP с аутентификацией CHAP, организации IP-связности между филиалами через сеть оператора связи с использованием статической и динамической маршрутизации.

### **Задачи**
1.  Построить топологию сети, включающую три филиала и сеть провайдера (три маршрутизатора).
2.  Настроить IP-адресацию на всех устройствах в соответствии с заданием.
3.  Настроить последовательные интерфейсы между маршрутизаторами филиалов и провайдера, используя инкапсуляцию PPP и аутентификацию CHAP.
4.  Организовать маршрутизацию внутри сети провайдера (с помощью протокола OSPF или статических маршрутов).
5.  Настроить маршруты по умолчанию на маршрутизаторах филиалов для доступа к другим филиалам через провайдера.
6.  Проверить связность между компьютерами в разных филиалах с помощью `ping` и `traceroute`.
7.  Проанализировать захваченный трафик (опционально) для наблюдения за работой PPP и маршрутизации.

### **Оборудование и программное обеспечение**
- **Программное обеспечение:** Cisco Packet Tracer
- **Оборудование (в эмуляторе):**
  - Маршрутизаторы Cisco 1941 (с модулями WIC-2T для последовательных интерфейсов) – 6 шт. (R1, R2, R3 – филиалы; PE1, PE2, PE3 – провайдер).
  - Коммутаторы Cisco 2960 – 3 шт. (по одному на филиал).
  - Персональные компьютеры – 6 шт. (по два в каждом филиале).
  - Соединительные провода: медные прямые (для подключения ПК к коммутаторам, коммутаторов к маршрутизаторам филиалов) и последовательные кабели (для соединений "филиал-провайдер" и между провайдерскими маршрутизаторами).

### **Краткие теоретические сведения**

**Глобальные сети (WAN)** предназначены для объединения локальных сетей, находящихся на значительном удалении друг от друга. В классических моделях сеть оператора связи (провайдера) предоставляет транспортные услуги для передачи данных между географически распределенными площадками заказчика.

**Типовые элементы архитектуры:**
- **CE (Customer Edge)** – оборудование на стороне клиента (маршрутизатор филиала).
- **PE (Provider Edge)** – оборудование на стороне провайдера, к которому подключается клиент.
- **P (Provider)** – внутренние маршрутизаторы сети провайдера.

**Инкапсуляция канального уровня на последовательных интерфейсах:**
- **HDLC (High-Level Data Link Control)** – протокол по умолчанию для последовательных интерфейсов Cisco. Не поддерживает аутентификацию.
- **PPP (Point-to-Point Protocol)** – более функциональный протокол, поддерживающий аутентификацию (PAP, CHAP), сжатие, многоканальность.

**Аутентификация CHAP (Challenge Handshake Authentication Protocol)** – трехэтапное рукопожатие, при котором пароль не передается в открытом виде, а используется для вычисления хэша. Настраивается на обоих концах соединения.

**Маршрутизация в подобных схемах:**
- Внутри сети провайдера обычно используется динамическая маршрутизация (OSPF, IS-IS) для обеспечения отказоустойчивости и масштабируемости.
- На клиентских маршрутизаторах часто достаточно настроить маршрут по умолчанию в сторону провайдера, а провайдер, в свою очередь, должен знать, как достичь сетей клиентов (через статические маршруты или через протоколы маршрутизации между CE и PE).

### **Порядок выполнения работы**

#### **1. Подготовительный этап**
1.  Запустите Cisco Packet Tracer.
2.  Добавьте на рабочее поле необходимое оборудование:
    - Маршрутизаторы: 6 шт. (1941).
    - Коммутаторы: 3 шт. (2960).
    - ПК: 6 шт.
3.  Подключите устройства согласно схеме (рисунок приложен к работе). Обратите внимание: для последовательных соединений используйте соответствующие интерфейсы (например, Serial0/1/0 или Serial0/0/0). При необходимости добавьте модули WIC-2T в слоты маршрутизаторов.

#### **2. Основной этап**

**Шаг 1. Настройка базовых параметров маршрутизаторов**
- Присвойте имена устройствам согласно схеме.
- Отключите DNS lookup: `no ip domain-lookup`.

**Шаг 2. Настройка IP-адресации на интерфейсах филиалов и провайдера**

*Пример для R1 (Филиал А):*
```
interface FastEthernet0/0
 ip address 192.168.1.1 255.255.255.0
 no shutdown
!
interface Serial0/1/0
 ip address 10.1.1.2 255.255.255.252
 encapsulation ppp
 ppp chap hostname R1
 ppp chap password cisco
 no shutdown
```
Аналогично настройте R2, R3, заменяя адреса и имена (для CHAP пароль везде можно сделать одинаковым, например "cisco").

*Настройка PE1:*
```
interface Serial0/1/0
 ip address 10.1.1.1 255.255.255.252
 encapsulation ppp
 ppp chap hostname PE1
 ppp chap password cisco
 no shutdown
!
interface Serial0/0/0
 ip address 172.16.0.1 255.255.255.252
 no shutdown
!
interface Loopback0
 ip address 1.1.1.1 255.255.255.255
!
```
Настройте PE2 и PE3 аналогично, учитывая адресацию из схемы:
- PE2: Serial0/1/0 (к R2) – 10.2.2.1/30, Serial0/0/0 (к PE1) – 172.16.0.2/30, Serial0/1/1 (к PE3) – 172.16.1.1/30.
- PE3: Serial0/1/0 (к R3) – 10.3.3.1/30, Serial0/0/0 (к PE2) – 172.16.1.2/30.

**Шаг 3. Проверка работы PPP и CHAP**
- После настройки интерфейсов проверьте состояние линии: `show interfaces serial0/1/0`. Статус должен быть "up/up".
- Проверьте аутентификацию: `debug ppp authentication` (включите на короткое время, затем отключите).

**Шаг 4. Настройка маршрутизации внутри сети провайдера**
На всех провайдерских маршрутизаторах (PE1, PE2, PE3) настройте OSPF в для обмена информацией о сетях между ними, включая loopback-адреса и сети подключения клиентов (чтобы провайдер знал, как достичь сетей филиалов).

*Пример для PE1:*
```
router ospf 1
 router-id 1.1.1.1
 network 172.16.0.0 0.0.0.3 area 0
 network 172.16.1.0 0.0.0.3 area 0
 network 10.1.1.0 0.0.0.3 area 0
 network 1.1.1.1 0.0.0.0 area 0
```
Аналогично для PE2 и PE3, включая свои сети.

**Шаг 5. Настройка маршрутов по умолчанию на филиалах**
Чтобы трафик из филиалов в другие сети проходил через провайдера, на R1, R2, R3 настроим маршрут по умолчанию в сторону своего PE.
```
ip route 0.0.0.0 0.0.0.0 10.1.1.1
```

**Шаг 6. Настройка маршрутов к сетям филиалов на провайдерских маршрутизаторах**
На данный момент провайдер ничего не знает о сетях внутри филиалов (192.168.x.0/24). Придумайте несколько вариантов, как добиться маршрутизации трафика между филиалами. Реализуйте наиболее подходящий по вашему мнению способ.

**Шаг 7. Настройка клиентских ПК**
- PC1: IP 192.168.1.10/24, шлюз 192.168.1.1
- PC2: IP 192.168.1.11/24, шлюз 192.168.1.1
- PC3: IP 192.168.2.10/24, шлюз 192.168.2.1
- PC4: IP 192.168.2.11/24, шлюз 192.168.2.1
- PC5: IP 192.168.3.10/24, шлюз 192.168.3.1
- PC6: IP 192.168.3.11/24, шлюз 192.168.3.1

#### **3. Контрольный этап**

2.  Проверьте связность между филиалами:
    - PC1 (192.168.1.10) до PC3 (192.168.2.10).
    - PC1 до PC5 (192.168.3.10).
    - PC3 до PC5.
3.  Выполните `traceroute` от PC1 до PC5, проанализируйте путь.
