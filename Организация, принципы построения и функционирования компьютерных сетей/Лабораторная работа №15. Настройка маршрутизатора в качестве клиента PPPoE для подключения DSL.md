### 1. Тема
Настройка маршрутизатора Cisco в качестве PPPoE-клиента для подключения к сети провайдера по технологии DSL.

### 2. Цель работы
- **Теоретическая:** Изучить принципы работы протокола PPPoE (Point-to-Point Protocol over Ethernet), этапы установления PPPoE-соединения (Discovery и Session), механизмы аутентификации CHAP/PAP и особенности MTU в PPPoE-сетях.
- **Практическая:** Приобрести навыки настройки маршрутизатора Cisco в качестве PPPoE-клиента, настройки PPPoE-сервера на стороне провайдера, организации маршрутизации и проверки работоспособности соединения.

### 3. Задачи
1. Построить топологию сети, включающую три филиала с PPPoE-подключением к центральному офису провайдера.
2. Настроить PPPoE-сервер на маршрутизаторе провайдера (создание пула IP-адресов, BBA-группы, виртуального шаблона).
3. Настроить три маршрутизатора филиалов в качестве PPPoE-клиентов (создание dialer-интерфейса, настройка CHAP-аутентификации).
4. Настроить маршрутизацию между филиалами через сеть провайдера.
5. Настроить трансляцию сетевых адресов (NAT) для доступа клиентских ПК в Internet.
6. Проверить связность между клиентскими ПК в разных филиалах и доступ к внешним ресурсам.
7. Выполнить захват и анализ PPPoE-трафика для наблюдения этапов установления соединения.

### 4. Оборудование и программное обеспечение
- **Программное обеспечение:** Cisco Packet Tracer 8.x.
- **Оборудование (в эмуляторе):**
  - Маршрутизаторы Cisco 1941: 5 шт. (ISP – провайдер, BR1 – филиал №1, BR2 – филиал №2, BR3 – филиал №3, EXT – внешний маршрутизатор для имитации Internet)
  - Коммутаторы Cisco 2960: 4 шт. (по одному в каждом филиале + для подключения внешней сети)
  - Персональные компьютеры: 6 шт. (по два в каждом филиале)
  - Сервер (для имитации web-сервера в Internet)
  - Соединительные провода: медные прямые (straight-through) для подключения ПК и серверов.

### 5. Краткие теоретические сведения

**PPPoE (Point-to-Point Protocol over Ethernet)** – протокол канального уровня, инкапсулирующий PPP-кадры в Ethernet-кадры. Широко используется провайдерами для предоставления доступа в Интернет по технологиям DSL (Digital Subscriber Line). PPPoE сочетает экономичность Ethernet-доступа с возможностями управления, аутентификации и учёта трафика, присущими PPP[reference:0].

**Процесс установления PPPoE-соединения проходит два этапа:**

**1. Discovery Phase (этап обнаружения):** В этом этапе клиент находит PPPoE-сервер в сети. Процесс включает обмен четырьмя типами пакетов[reference:1]:
- **PADI (PPPoE Active Discovery Initiation):** Клиент широковещательно рассылает запрос на поиск сервера.
- **PADO (PPPoE Active Discovery Offer):** Сервер(ы) отвечают предложением услуг.
- **PADR (PPPoE Active Discovery Request):** Клиент выбирает один сервер и отправляет запрос на установку сессии.
- **PADS (PPPoE Active Discovery Session-confirmation):** Сервер подтверждает создание сессии и назначает уникальный Session ID.

**2. PPP Session Phase (сеансовый этап):** После успешного Discovery начинается стандартная PPP-сессия, включающая три подэтапа[reference:2]:
- **LCP (Link Control Protocol):** Установка и конфигурирование канала.
- **Аутентификация (CHAP/PAP):** Проверка учётных данных клиента.
- **NCP (Network Control Protocol):** Настройка сетевых параметров (например, IPCP для назначения IP-адреса).

**MTU в PPPoE:** Стандартный MTU Ethernet составляет 1500 байт. При использовании PPPoE добавляются заголовки PPP (2 байта) и PPPoE (6 байт), что снижает максимальный размер полезной нагрузки до 1492 байт[reference:3]. Для корректной работы TCP-приложений на dialer-интерфейсе рекомендуется настраивать `ip mtu 1492` и `ip tcp adjust-mss 1452`[reference:4].

#### 6.2. Основной этап

##### Шаг 1. Настройка базовых параметров маршрутизаторов и коммутаторов
- Присвойте имена устройствам согласно схеме.
- Отключите DNS lookup: `no ip domain-lookup`.
- Настройте пароли для доступа (при необходимости).
- Настройте на коммутаторах SW1, SW2, SW3 VLAN 1 и порты доступа для клиентских ПК.

##### Шаг 2. Настройка маршрутизатора провайдера (ISP)
ISP выполняет роль PPPoE-сервера и предоставляет доступ трём филиалам через три различных VLAN.

**Настройка ISP:**
```
enable
configure terminal
hostname ISP
!
! Настройка интерфейса во внешнюю сеть
interface FastEthernet0/1
 ip address 200.0.0.1 255.255.255.0
 no shutdown
!
! Создание VLAN-субинтерфейсов для подключения филиалов
interface FastEthernet0/0
 no ip address
 no shutdown
!
interface FastEthernet0/0.10
 encapsulation dot1Q 10
!
interface FastEthernet0/0.20
 encapsulation dot1Q 20
!
interface FastEthernet0/0.30
 encapsulation dot1Q 30
!
! Настройка PPPoE для филиала BR1 (VLAN 10)
username BR1 password cisco123
ip local pool PPPoE_Pool_BR1 10.0.10.10 10.0.10.20
bba-group pppoe GROUP_BR1
 virtual-template 1
!
interface Virtual-Template1
 ip unnumbered FastEthernet0/0.10
 peer default ip address pool PPPoE_Pool_BR1
 ppp authentication chap
!
interface FastEthernet0/0.10
 encapsulation dot1Q 10
 pppoe enable group GROUP_BR1
 no shutdown
!
! Настройка PPPoE для филиала BR2 (VLAN 20)
username BR2 password cisco123
ip local pool PPPoE_Pool_BR2 10.0.20.10 10.0.20.20
bba-group pppoe GROUP_BR2
 virtual-template 2
!
interface Virtual-Template2
 ip unnumbered FastEthernet0/0.20
 peer default ip address pool PPPoE_Pool_BR2
 ppp authentication chap
!
interface FastEthernet0/0.20
 encapsulation dot1Q 20
 pppoe enable group GROUP_BR2
 no shutdown
!
! Настройка PPPoE для филиала BR3 (VLAN 30)
username BR3 password cisco123
ip local pool PPPoE_Pool_BR3 10.0.30.10 10.0.30.20
bba-group pppoe GROUP_BR3
 virtual-template 3
!
interface Virtual-Template3
 ip unnumbered FastEthernet0/0.30
 peer default ip address pool PPPoE_Pool_BR3
 ppp authentication chap
!
interface FastEthernet0/0.30
 encapsulation dot1Q 30
 pppoe enable group GROUP_BR3
 no shutdown
!
! Статический маршрут по умолчанию во внешнюю сеть
ip route 0.0.0.0 0.0.0.0 200.0.0.2
!
! Маршруты обратно к клиентским сетям (для доступа клиентов друг к другу)
ip route 192.168.1.0 255.255.255.0 10.0.10.10
ip route 192.168.2.0 255.255.255.0 10.0.20.10
ip route 192.168.3.0 255.255.255.0 10.0.30.10
!
end
write memory
```

##### Шаг 3. Настройка маршрутизатора филиала (на примере BR1)
```
enable
configure terminal
hostname BR1
!
! Настройка LAN-интерфейса
interface FastEthernet0/1
 ip address 192.168.1.1 255.255.255.0
 no shutdown
!
! Настройка PPPoE-клиента на WAN-интерфейсе
interface FastEthernet0/0
 no ip address
 pppoe-client dial-pool-number 1
 no shutdown
!
! Настройка dialer-интерфейса
interface Dialer1
 mtu 1492
 ip address negotiated
 encapsulation ppp
 dialer pool 1
 ppp chap hostname BR1
 ppp chap password cisco123
 ip tcp adjust-mss 1452
 no shutdown
!
! Маршрут по умолчанию через dialer-интерфейс
ip route 0.0.0.0 0.0.0.0 Dialer1
!
end
write memory
```

**Аналогично настройте BR2 и BR3** с соответствующими IP-адресами LAN-интерфейсов и учётными данными:
- BR2: LAN 192.168.2.1/24, username BR2, password cisco123
- BR3: LAN 192.168.3.1/24, username BR3, password cisco123

##### Шаг 4. Настройка маршрутизатора внешней сети (EXT) и серверов
```
enable
configure terminal
hostname EXT
!
interface FastEthernet0/0

!
interface FastEthernet0/1

!
end
write memory
```

##### **Настройка маршрутизации и NAT**
1.  Настройте на BR1, BR2, BR3 трансляцию адресов (NAT) для доступа локальных ПК в Internet. Используйте перегрузку портов (PAT) на dialer-интерфейсе.
2.  Настройте на ISP маршрутизацию с использованием протокола OSPF вместо статических маршрутов для распространения информации о клиентских сетях между филиалами.

#### 6.3. Контрольный этап
1. **Проверка установления PPPoE-сессии:**
   ```
   show ip interface brief
   show dialer
   show pppoe session
   show interface dialer 1
   ```
   Убедитесь, что Dialer1 получил IP-адрес (ip address negotiated).

2. **Проверка маршрутизации:**
   ```
   show ip route
   show ip route ospf
   ```
   На BR1 должен присутствовать маршрут по умолчанию 0.0.0.0/0 через Dialer1.

3. **Проверка связности:**
   - С PC1 выполните ping до PC3 (192.168.2.10)
   - С PC1 выполните ping до PC5 (192.168.3.10)
   - С PC1 выполните ping до Web-сервера (209.165.200.225)
   - С PC1 выполните ping до DNS-сервера (8.8.8.8)

4. **Проверка разрешения имён:**
   ```
   nslookup www.example.com 8.8.8.8
   ```
   Зайти в браузере с PC1 на www.example.com

5. **Включите симуляцию в Packet Tracer и проанализируйте PPPoE-трафик:**
   - Перейдите в режим Simulation.
   - Добавьте фильтр для протокола PPPoE.
   - Запустите захват и наблюдайте последовательность PADI → PADO → PADR → PADS.

6. **Сохраните конфигурации на всех маршрутизаторах:**
   ```
   copy running-config startup-config
   ```
