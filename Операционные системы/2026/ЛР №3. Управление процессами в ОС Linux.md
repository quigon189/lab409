### Тема
Управление процессами в операционной системе Linux

### Цель работы
- **Теоретическая:** Изучить базовые и продвинутые методы управления процессами в Linux, включая мониторинг, изменение приоритетов, завершение процессов и работу с системными службами.
- **Практическая:** Приобрести навыки использования команд `ps`, `top`, `htop`, `kill`, `systemctl`, работы с псевдофайловой системой `/proc` и управления фоновыми процессами.

### Задачи
1. Освоить мониторинг процессов с помощью команд `ps`, `top` и утилиты `htop`.
2. Научиться управлять процессами: запуск в фоне, приостановка, возобновление, завершение с использованием сигналов.
3. Познакомиться с системой инициализации `systemd` и научиться управлять системными службами.
4. Изучить информацию о процессах, доступную в псевдофайловой системе `/proc`.

### Оборудование и программное обеспечение
- **Программное обеспечение:** Операционная система Debian 10 (или любой дистрибутив Linux с ядром, поддерживающим `systemd`).
- **Необходимые пакеты:** `procps` (содержит `ps`, `kill`), `htop`, `systemd` (установлен по умолчанию).
- **Оборудование (виртуальное/реальное):** Персональный компьютер или виртуальная машина с ОС Linux, доступ к терминалу и права `sudo` для управления службами.

### Краткие теоретические сведения

**Процесс** в Linux — это экземпляр выполняющейся программы, которому присваивается уникальный идентификатор (PID). Каждый процесс имеет родителя (PPID). Процессы могут работать в фоновом режиме, на переднем плане, быть приостановленными или завершёнными.

**Сигналы** — способ асинхронного уведомления процессов о событиях. Основные сигналы:
- `SIGTERM` (15) — вежливый запрос на завершение (процесс может перехватить и обработать).
- `SIGKILL` (9) — безусловное принудительное завершение (не перехватывается).
- `SIGSTOP` (19) — приостановка выполнения.
- `SIGCONT` (18) — возобновление приостановленного процесса.

**Приоритеты процессов** определяются значением `nice`: от -20 (самый высокий приоритет) до 19 (самый низкий). По умолчанию процессы запускаются с приоритетом 0. Изменить приоритет можно командами `nice` и `renice`.

**Система инициализации** `systemd` является стандартом в большинстве современных дистрибутивов. Она управляет запуском служб (юнитов) и их состоянием. Основные команды: `systemctl start|stop|restart|status|enable|disable`.

**Псевдофайловая система `/proc`** предоставляет интерфейс для доступа к информации о процессах и ядре. Для каждого процесса существует каталог `/proc/<PID>`, содержащий файлы:
- `cmdline` — командная строка запуска.
- `environ` — переменные окружения.
- `status` — статус процесса (PID, PPID, состояние, использование памяти и др.).
- `fd/` — ссылки на открытые файловые дескрипторы.

### Порядок выполнения работы

> **Важно:** По каждому пункту работы необходимо сделать скриншот для последующего включения в отчёт.

#### 1. Подготовительный этап
1. Войдите в систему под своей учётной записью.
2. Откройте эмулятор терминала.

#### 2. Основной этап

##### **Задание 1: Мониторинг процессов**
1. **Команда `ps`:**  
   - Выведите список всех процессов текущего пользователя:
     ```bash
     ps -u $(whoami)
     ```
   - Просмотрите все процессы в системе с детализацией (формат BSD):
     ```bash
     ps aux
     ```
   - Найдите процесс `sshd` (демон SSH) с фильтрацией через `grep`:
     ```bash
     ps aux | grep sshd
     ```

2. **Утилита `top`:**  
   - Запустите `top` (интерактивный мониторинг).
   - Отсортируйте процессы по:
     - Потреблению CPU — нажмите клавишу `P`.
     - Потреблению памяти — нажмите клавишу `M`.
   - Запишите PID процесса, который создаёт наибольшую нагрузку (можно записать в отчёт).

3. **Утилита `htop`:**  
   - Запустите `htop` (если не установлена, установите: `sudo apt install htop`).
   - Используйте фильтр (`F4`) для поиска процессов, связанных с `bash`.
   - Выберите любой процесс `bash` и завершите его клавишей `F9` (подтвердите сигнал `SIGTERM`). Обратите внимание, что завершение терминального процесса может закрыть текущее окно, поэтому будьте осторожны.

##### **Задание 2: Управление процессами**
1. **Запуск и завершение:**  
   - Запустите процесс в фоне:
     ```bash
     sleep 600 &
     ```
   - Найдите его PID с помощью команды `jobs -l` (отображает фоновые задачи текущей сессии) или `ps`.
   - Завершите процесс, отправив сигнал `SIGTERM`:
     ```bash
     kill <PID>
     ```

2. **Сигналы:**  
   - Запустите текстовый редактор `nano test.txt`.
   - Приостановите его выполнение сочетанием клавиш `Ctrl+Z` (отправит сигнал `SIGTSTP`).
   - Переведите приостановленный процесс в фоновый режим:
     ```bash
     bg
     ```
   - Верните фоновый процесс на передний план:
     ```bash
     fg
     ```
   - Завершите редактор (обычно `Ctrl+X`).

##### **Задание 3: Работа с системными службами (systemd)**
1. **Просмотр служб:**  
   - Выведите список всех активных служб:
     ```bash
     systemctl list-units --type=service
     ```
   - Проверьте статус службы `ssh`:
     ```bash
     systemctl status ssh
     ```

2. **Управление службами:**  
   - Перезапустите службу SSH:
     ```bash
     sudo systemctl restart ssh
     ```

##### **Задание 4: Работа с /proc**
1. Перейдите в директорию процесса `init` (в современных системах `init` — символическая ссылка на `systemd`, PID 1):
   ```bash
   cd /proc/1
   ```
2. Просмотрите следующие файлы:
   - Командная строка запуска:
     ```bash
     cat cmdline
     ```
   - Переменные среды (много информации, можно использовать `more` или `less`):
     ```bash
     cat environ | tr '\0' '\n'   # для удобочитаемости
     ```
   - Статус процесса:
     ```bash
     cat status
     ```
