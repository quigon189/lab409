### Тема
Изучение основных конструкций командной оболочки Bash. Знакомство с Bash-скриптами.

### Цель работы
- **Теоретическая:** Изучить базовые конструкции Bash: переменные, условные операторы, циклы, функции; понять принципы работы с аргументами и вводом/выводом.
- **Практическая:** Приобрести навыки написания скриптов для автоматизации задач, научиться создавать файлы и каталоги, выполнять проверки, организовывать интерактивные меню и резервное копирование.

### Задачи
1. Изучить синтаксис объявления и использования переменных в Bash.
2. Освоить условные операторы (`if`, `case`) и операторы сравнения.
3. Научиться применять циклы (`for`, `while`) для повторяющихся действий.
4. Познакомиться с созданием и вызовом функций.
5. Написать несколько скриптов, демонстрирующих различные конструкции:
   - скрипт для создания каталога и файла с проверками;
   - интерактивный скрипт с выбором действия;
   - скрипт резервного копирования с использованием `tar`.
6. Изучить дополнительные возможности: комбинирование команд (`&&`, `||`), чтение пользовательского ввода, работа с датой.

### Оборудование и программное обеспечение
- **Программное обеспечение:** Любой дистрибутив Linux с установленной оболочкой Bash (по умолчанию в Ubuntu, CentOS, Fedora и др.), текстовый редактор (nano, vim, gedit).
- **Оборудование (виртуальное/реальное):** Персональный компьютер или виртуальная машина с ОС Linux.

### Краткие теоретические сведения

**Bash (Bourne Again Shell)** — командная оболочка и язык сценариев, стандартная в большинстве дистрибутивов Linux. Она позволяет выполнять команды как в интерактивном режиме, так и в виде скриптов (файлов с последовательностями команд).

**Переменные** в Bash создаются присваиванием без пробелов: `name="value"`. Для доступа к значению используется знак `$`: `echo $name`. Особые переменные: `$0` — имя скрипта, `$1`, `$2`... — позиционные аргументы, `$#` — количество аргументов, `$@` — все аргументы, `$?` — код возврата последней команды.

**Типы данных** строго не фиксированы: Bash оперирует строками, но поддерживает целочисленные арифметические операции через `$(( ... ))`.

**Условные операторы**:
- `if [ условие ]; then ... fi` — пробелы вокруг условия обязательны.
- Операции сравнения чисел: `-lt`, `-gt`, `-le`, `-ge`, `-eq`, `-ne`.
- Для файлов: `-d` (каталог), `-f` (файл), `-e` (существует) и др.
- `case ... in` — выбор по шаблону.

**Циклы**:
- `for var in list; do ... done` — перебор элементов.
- `for (( i=0; i<10; i++ )); do ... done` — арифметический цикл.
- `while [ условие ]; do ... done` — выполняется, пока условие истинно.
- `until [ условие ]; do ... done` — выполняется, пока условие ложно.

**Функции** определяются как `function_name() { ... }` или `function function_name { ... }`. Аргументы внутри функции доступны как `$1`, `$2` и т.д. Возврат значения осуществляется через `return` (числовой код) или `echo` для захвата вывода.

**Комбинирование команд**:
- `&&` — следующая команда выполняется только при успехе предыдущей.
- `||` — следующая команда выполняется только при неудаче предыдущей.

##### Задание 1. Создание первого скрипта
1. С помощью текстового редактора (например, `nano`) создайте файл `script1.sh`.
2. В первой строке укажите шебанг: `#!/bin/bash`.
3. Напишите скрипт, который принимает два аргумента: имя каталога и имя файла. Скрипт должен выводить сообщение "Создаю каталог <каталог>", создавать этот каталог, а затем создавать внутри него указанный файл.
   ```bash
   #!/bin/bash
   dir_name=$1
   file_name=$2
   echo "Создаю каталог <$dir_name>"
   mkdir "$dir_name"
   echo "Создаю в каталоге <$dir_name> файл <$file_name>"
   touch "$dir_name/$file_name"
   exit 0
   ```
4. Выполните скрипт двумя способами:
   - `bash script1.sh test_dir test1.txt`
   - Сделайте скрипт исполняемым: `chmod +x script1.sh` и запустите `./script1.sh test_dir test2.txt`
5. Проанализируйте результат. Обратите внимание на то, что происходит при повторном запуске с тем же именем каталога.

##### Задание 2. Добавление проверки существования каталога
1. Модифицируйте скрипт `script1.sh`, добавив проверку на существование каталога перед созданием. Используйте условие `if [ -d "$dir_name" ]`.
   ```bash
   #!/bin/bash
   dir_name=$1
   file_name=$2
   if [ -d "$dir_name" ]; then
       echo "Каталог с таким именем уже существует"
   else
       echo "Создаю каталог <$dir_name>"
       mkdir "$dir_name"
   fi
   touch "$dir_name/$file_name" 2>/dev/null && echo "Файл создан" || echo "Не удалось создать файл"
   ```
2. Запустите скрипт несколько раз с одинаковыми и разными именами каталогов, наблюдайте за поведением.

##### Задание 3. Проверка на существование файла
1. Добавьте проверку на существование файла перед его созданием. Используйте опцию `-f`. Выведите соответствующее сообщение, если файл уже есть.
   ```bash
   #!/bin/bash
   dir_name=$1
   file_name=$2
   if [ -d "$dir_name" ]; then
       echo "Каталог существует"
   else
       mkdir "$dir_name" && echo "Каталог создан"
   fi
   if [ -f "$dir_name/$file_name" ]; then
       echo "Файл уже существует"
   else
       touch "$dir_name/$file_name" && echo "Файл создан"
   fi
   ```

##### Задание 4. Интерактивный скрипт с выбором действия
1. Создайте новый скрипт `interactive.sh`, который предлагает пользователю выбрать, что создать: каталог или файл. Используйте `echo` для вывода меню, `read` для ввода, и `case` для обработки выбора.
   ```bash
   #!/bin/bash
   echo "Выберите, что будем создавать:"
   echo "1 – создать каталог"
   echo "2 – создать файл"
   echo -n "Ваш выбор: "
   read choice
   case $choice in
       1)
           echo -n "Введите название каталога: "
           read dir
           mkdir "$dir" && echo "Каталог создан" || echo "Не удалось создать каталог"
           ;;
       2)
           echo -n "Введите название файла: "
           read file
           touch "$file" && echo "Файл создан" || echo "Не удалось создать файл"
           ;;
       *)
           echo "Введено значение отличное от 1 или 2"
           ;;
   esac
   exit 0
   ```
2. Протестируйте скрипт, создайте каталог с номером вашей группы и файл с вашей фамилией. Попробуйте создать каталог повторно.

##### Задание 5. Скрипт резервного копирования
1. Изучите команду `tar` с помощью `man tar`. Обратите внимание на ключи `-c` (создать архив), `-f` (задать имя файла), `-z` (сжатие gzip).
2. Напишите скрипт `backup.sh`, который:
   - запрашивает у пользователя путь к каталогу для резервирования;
   - запрашивает путь для сохранения резервной копии;
   - спрашивает, нужно ли архивировать каталог (да/нет);
   - в зависимости от ответа либо копирует каталог целиком (`cp -r`), либо создаёт архив с именем, содержащим текущую дату (используйте `date +%Y%m%d`).
   Пример:
   ```bash
   #!/bin/bash
   read -p "Введите путь к каталогу для резервирования: " source_dir
   read -p "Введите путь для сохранения резервной копии: " dest_dir
   read -p "Архивировать? (y/n): " answer
   if [ ! -d "$source_dir" ]; then
       echo "Ошибка: исходный каталог не существует"
       exit 1
   fi
   mkdir -p "$dest_dir"
   base_name=$(basename "$source_dir")
   date_str=$(date +%Y%m%d_%H%M%S)
   if [[ "$answer" == "y" || "$answer" == "Y" ]]; then
       archive_name="$dest_dir/${base_name}_backup_$date_str.tar.gz"
       tar -czf "$archive_name" -C "$(dirname "$source_dir")" "$base_name"
       echo "Архив создан: $archive_name"
   else
       cp -r "$source_dir" "$dest_dir/${base_name}_copy_$date_str"
       echo "Каталог скопирован в $dest_dir"
   fi
   ```
3. Проверьте работоспособность скрипта на любом тестовом каталоге.

##### Задание 6 (дополнительное). Многофункциональное меню
1. Создайте скрипт `menu.sh`, который выводит меню со следующими пунктами:
   - текущий пользователь;
   - объём используемой памяти;
   - системная дата и время;
   - время запуска системы;
   - выход.
2. Реализуйте бесконечный цикл (`while true`), обрабатывающий выбор пользователя с помощью `case`. Используйте команды: `whoami`, `free -h`, `date`, `uptime -s` (или `who -b`).
   ```bash
   #!/bin/bash
   while true; do
       echo "===================="
       echo "1. Текущий пользователь"
       echo "2. Объём используемой памяти"
       echo "3. Системная дата и время"
       echo "4. Время запуска системы"
       echo "5. Выход"
       echo -n "Выберите пункт: "
       read choice
       case $choice in
           1) whoami ;;
           2) free -h ;;
           3) date ;;
           4) uptime -s ;;
           5) echo "Выход"; break ;;
           *) echo "Неверный выбор" ;;
       esac
   done
   ```
