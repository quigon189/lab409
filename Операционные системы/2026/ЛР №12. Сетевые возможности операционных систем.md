### Тема
Изучение сетевых возможностей операционных систем Windows 10 и Manjaro Linux: идентификация служб стека TCP/IP, управление их запуском, статическая настройка IP-адреса и проверка связности.

### Цель работы
- **Теоретическая:** Изучить состав и назначение системных служб, реализующих стек протоколов TCP/IP в Windows и Linux; понять принципы статической и динамической (DHCP) настройки сетевых параметров.
- **Практическая:** Приобрести навыки поиска и управления сетевыми службами, изменения IP-адреса, маски подсети, шлюза по умолчанию, проверки работоспособности сети с помощью `ping`, восстановления исходной конфигурации.

### Задачи
1. **В Windows 10:**
   - Найти службы, отвечающие за работу TCP/IP (клиент DHCP, клиент DNS, служба сетевого расположения и др.).
   - Определить их статус и тип запуска.
   - Запомнить текущие настройки сетевого адаптера (IP, маска, шлюз, DNS).
   - Изменить конфигурацию на статическую (выбрать IP из одной подсети с соседом).
   - Проверить связь с соседом командой `ping`.
   - Вернуть исходные настройки (DHCP).

2. **В Linux Manjaro:**
   - Найти службы, управляющие сетью (NetworkManager, systemd-networkd, dhcpcd и др.).
   - Определить, какая служба активна.
   - Запомнить текущие настройки сетевого интерфейса.
   - Настроить статический IP через `nmcli` (или `ip` и `systemd-networkd`).
   - Проверить связь с соседом `ping`.
   - Вернуть исходные настройки (DHCP).

### Оборудование и программное обеспечение
- **Программное обеспечение:**
  - Windows 10 (Pro/Enterprise) с правами администратора.
  - Manjaro Linux (актуальная версия) с правами `sudo`.
- **Оборудование:** Два компьютера (или виртуальные машины) в одной локальной сети (физической или виртуальной), соединённые через коммутатор или напрямую (crossover). Студенты работают в парах, договариваясь о статических IP-адресах из одной подсети (например, 192.168.100.10 и 192.168.100.11, маска 255.255.255.0).

### Краткие теоретические сведения

**Стек протоколов TCP/IP** – основа современных сетей. В каждой операционной системе работа стека обеспечивается набором системных служб и драйверов.

**В Windows:**
- **DHCP Client** – служба, отвечающая за автоматическое получение IP-адреса от DHCP-сервера.
- **DNS Client** – служба кэширования DNS-запросов.
- **Network Location Awareness (NLA)** – определяет тип сети (частная, общественная) и влияет на параметры брандмауэра.
- **Network List Service** – отображает сетевые подключения в интерфейсе.
- Эти службы запускаются автоматически при загрузке. Управление через `services.msc` или PowerShell.

**В Linux (Manjaro):**
- **NetworkManager** – стандартный сервис для управления сетями в дистрибутивах с графическим окружением (GNOME, KDE, Xfce). Управление через `nmcli`, `nmtui` или графический апплет.
- **systemd-networkd** – более легковесный сервис от systemd, используется в серверных конфигурациях.
- **dhcpcd** – классический DHCP-клиент (может использоваться вместо NetworkManager).
- Посмотреть активные службы: `systemctl status NetworkManager`, `systemctl status systemd-networkd`.
- Статическая настройка возможна через `nmcli con mod "имя" ipv4.addresses ...`.

**Статическая настройка IP:**
- Адрес должен быть уникальным в пределах одной подсети.
- Маска подсети определяет, какие адреса считаются локальными.
- Шлюз по умолчанию – маршрутизатор для выхода в другие сети (при отсутствии роутера можно указать адрес соседа или оставить пустым для проверки только между соседями).
- DNS-серверы – для преобразования имён (можно указать публичные, например 8.8.8.8).

**Команда `ping`** – отправляет ICMP-запросы и проверяет связность с удалённым узлом.

### Порядок выполнения работы

> **Важно:** По каждому пункту работы необходимо сделать скриншот.  
> **Работа выполняется в парах.** Студенты договариваются о статических IP-адресах, которые не конфликтуют с другими устройствами в сети. Рекомендуется использовать адреса из диапазона 192.168.100.10–192.168.100.20 с маской /24 (255.255.255.0).

## Часть 1. Windows 10

#### 1. Подготовительный этап
1. Войдите в Windows 10 под учётной записью администратора.
2. Убедитесь, что ваш компьютер подключён к локальной сети (есть связь с соседом).

#### 2. Основной этап (Windows)

##### **Задание 1.1: Определение сетевых служб TCP/IP**
1. Откройте оснастку **Службы**:
   - Нажмите `Win + R`, введите `services.msc`, Enter.
2. Найдите следующие службы (используйте поиск по списку или сортировку по имени):
   - **DHCP Client**
   - **DNS Client**
   - **Network Location Awareness (NLA)**
   - **Network List Service**
3. Для каждой службы запишите:
   - **Статус** (Работает / Остановлена)
   - **Тип запуска** (Автоматически / Вручную / Отключена)
4. Выберите одну из служб (например, DHCP Client), откройте её свойства (двойной клик). Изучите вкладки «Общие», «Вход в систему», «Восстановление».
5. Сделайте скриншот окна «Службы» с выделенными службами и окна свойств одной из них.

##### **Задание 1.2: Запоминание текущей конфигурации**
1. Откройте **Центр управления сетями и общим доступом**:
   - ПКМ по значку сети в трее → «Центр управления сетями...»
   - Или `Панель управления` → `Сеть и Интернет` → `Центр управления сетями`.
2. Нажмите на имя вашего подключения (например, «Ethernet» или «Wi-Fi»).
3. В открывшемся окне нажмите **Сведения...**.
4. Запишите или сфотографируйте:
   - **IPv4-адрес**
   - **Маска подсети**
   - **Основной шлюз**
   - **DNS-серверы**
5. Сделайте скриншот окна «Сведения о сетевом подключении».

##### **Задание 1.3: Настройка статического IP-адреса (договоритесь с соседом)**
1. Вернитесь в окно состояния подключения, нажмите **Свойства**.
2. В списке компонентов выберите **IP версии 4 (TCP/IPv4)** → **Свойства**.
3. Переключитесь на **Использовать следующий IP-адрес**.
4. Введите параметры (согласуйте с соседом, чтобы адреса были из одной подсети и не совпадали):
   - **IP-адрес:** например, `192.168.100.10` (вы) и `192.168.100.11` (сосед)
   - **Маска подсети:** `255.255.255.0`
   - **Основной шлюз:** можно указать адрес соседа или оставить пустым (если нет роутера)
   - **Предпочитаемый DNS-сервер:** `8.8.8.8`
   - **Альтернативный DNS-сервер:** `8.8.4.4`
5. Нажмите **ОК** → **Закрыть**.
6. Сделайте скриншот окна свойств TCP/IPv4 с введёнными статическими адресами.

##### **Задание 1.4: Проверка связности**
1. Откройте командную строку (`Win + R` → `cmd` → Enter).
2. Выполните команду:
   ```cmd
   ping <IP-адрес_соседа>
   ```
   Например: `ping 192.168.100.11`
3. Если настройки верны и сеть исправна, вы должны увидеть ответы `Ответ от ...`.
4. Сделайте скриншот успешного выполнения `ping`.
5. При необходимости проверьте также доступ к интернету (если шлюз указан верно): `ping 8.8.8.8`.

##### **Задание 1.5: Возврат к исходной конфигурации**
1. Снова откройте свойства **IP версии 4 (TCP/IPv4)**.
2. Переключитесь на **Получить IP-адрес автоматически** и **Получить адрес DNS-сервера автоматически**.
3. Нажмите **ОК** → **Закрыть**.
4. Проверьте, что адрес вернулся к DHCP (можно заново открыть «Сведения»).
5. Сделайте скриншот, подтверждающий возврат к автоматической настройке.

## Часть 2. Linux Manjaro

#### 1. Подготовительный этап
1. Запустите Manjaro, откройте терминал.
2. Убедитесь, что у вас есть права `sudo`.

#### 2. Основной этап (Linux)

##### **Задание 2.1: Определение сетевых служб**
1. Проверьте, какие службы управления сетью активны:
   ```bash
   systemctl status NetworkManager
   systemctl status systemd-networkd
   systemctl status dhcpcd
   ```
   (Возможно, активна только одна из них; в Manjaro обычно используется NetworkManager.)
2. Запишите, какая служба имеет статус `active (running)` и `enabled`.
3. Для активной службы просмотрите её логи (например, для NetworkManager):
   ```bash
   journalctl -u NetworkManager --since "5 minutes ago" | tail -20
   ```
4. Сделайте скриншот вывода `systemctl status NetworkManager` и команды `journalctl`.

##### **Задание 2.2: Запоминание текущей конфигурации**
1. Определите имя вашего сетевого интерфейса:
   ```bash
   ip a
   ```
   Обычно это `eth0`, `enpXsY` или `wlan0`.
2. Запишите текущие настройки (IPv4, маска, шлюз, DNS):
   ```bash
   ip -4 addr show <интерфейс>
   ip route show default
   # DNS-серверы (если используете NetworkManager)
   nmcli device show <интерфейс> | grep DNS
   ```
   Либо, если NetworkManager не используется, посмотрите `/etc/resolv.conf`.
3. Сделайте скриншот вывода команд `ip a` и `nmcli device show`.

##### **Задание 2.3: Настройка статического IP-адреса (через nmcli)**

1. Посмотрите список подключений:
   ```bash
   nmcli con show
   ```
   Запомните имя активного подключения (например, `Wired connection 1`).
2. Измените подключение на статическое (замените `Wired connection 1` на ваше имя, IP и шлюз – договоритесь с соседом):
   ```bash
   sudo nmcli con mod "Wired connection 1" ipv4.method manual ipv4.addresses 192.168.100.10/24 ipv4.gateway 192.168.100.1
   sudo nmcli con mod "Wired connection 1" ipv4.dns "8.8.8.8 8.8.4.4"
   ```
   *Если шлюза нет (прямое подключение к соседу), можно опустить `ipv4.gateway` или указать адрес соседа.*
3. Перезапустите подключение:
   ```bash
   sudo nmcli con down "Wired connection 1" && sudo nmcli con up "Wired connection 1"
   ```
4. Проверьте новые настройки:
   ```bash
   ip a show <интерфейс>
   ```
5. Сделайте скриншот введённых команд и вывода `ip a`.

##### **Задание 2.4: Проверка связности**
1. Выполните `ping` на соседа:
   ```bash
   ping -c 4 <IP-адрес_соседа>
   ```
2. Если `ping` не проходит, проверьте:
   - Правильно ли указана маска подсети.
   - Отключён ли брандмауэр на обеих машинах (временно `sudo systemctl stop firewalld` или `sudo ufw disable`).
   - Находятся ли компьютеры в одной подсети.
3. Сделайте скриншот успешного `ping`.

##### **Задание 2.5: Возврат к исходной конфигурации (DHCP)**
1. Верните начальную конфигурацию.
2. Убедитесь, что конфигурация верна:
   ```bash
   ip a show <интерфейс>
   ```
3. Сделайте скриншот вывода `ip a` после возврата.
#### 3. Контрольный этап
1. Убедитесь, что в обеих ОС исходные настройки восстановлены.
2. Подготовьте ответы на контрольные вопросы.
