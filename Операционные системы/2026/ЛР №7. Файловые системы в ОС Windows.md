### Тема
Изучение файловых систем, их характеристик и инструментов управления в Windows.

### Цель работы
- **Теоретическая:** Изучить принципы работы файловых систем NTFS, FAT32 и exFAT, их особенности, преимущества и ограничения.
- **Практическая:** Приобрести навыки создания разделов, форматирования дисков в разные файловые системы, настройки прав доступа, сжатия данных, диагностики и восстановления файловой системы с помощью встроенных инструментов Windows.

### Задачи
1. Создать виртуальный жесткий диск (VHD) через оснастку «Управление дисками» и командную строку (diskpart).
2. Отформатировать разделы виртуального диска в файловые системы NTFS, FAT32 и exFAT.
3. Изучить возможности NTFS: настройка прав доступа, сжатие данных.
4. Проверить ограничения FAT32 (максимальный размер файла).
5. Выполнить диагностику и восстановление файловой системы с помощью `chkdsk`.
6. Ознакомиться с восстановлением удалённых файлов с помощью утилиты Recuva.

### Оборудование и программное обеспечение
- **Программное обеспечение:** Операционная система Windows 10 (любая редакция) с правами администратора.
- **Дополнительное ПО:** Утилита Recuva (portable версия, предоставленная преподавателем или скачанная с официального сайта).
- **Оборудование (виртуальное/реальное):** Персональный компьютер или виртуальная машина с Windows 10, достаточным объёмом свободного места на диске (минимум 10 ГБ для создания VHD).

### Краткие теоретические сведения

**Файловая система** определяет способ организации, хранения и именования данных на носителе. В Windows наиболее распространены:

- **NTFS (New Technology File System)** — стандартная файловая система для Windows NT и более поздних версий. Поддерживает: журналирование (устойчивость к сбоям), права доступа (ACL), шифрование (EFS), сжатие, квоты, теневые копии, жёсткие ссылки, символические ссылки, максимальный размер файла и тома до 256 ТБ (теоретически до 16 ЭБ).
- **FAT32 (File Allocation Table 32)** — устаревшая, но широко совместимая ФС. Ограничения: максимальный размер файла 4 ГБ, максимальный размер тома 2 ТБ (при 512-байтовых секторах). Не поддерживает права доступа, журналирование, сжатие.
- **exFAT (Extended File Allocation Table)** — разработана для флеш-накопителей. Снимает ограничение FAT32 на размер файла (до 16 ЭБ), поддерживает большие тома, но сохраняет простоту. Не поддерживает права доступа и журналирование.

**Создание разделов** в Windows выполняется через оснастку `diskmgmt.msc` (Управление дисками) или утилиту командной строки `diskpart`. Виртуальные жесткие диски (VHD) позволяют эмулировать физический диск в файле, что удобно для лабораторных работ.

**Диагностика ФС:** `chkdsk` (Check Disk) — проверяет целостность файловой системы и исправляет логические ошибки (ключ `/f`), а также обнаруживает физические повреждённые сектора (ключ `/r`).

**Восстановление данных:** удалённые файлы не уничтожаются немедленно, а помечаются как свободное место. Специализированные утилиты (Recuva, TestDisk) могут восстановить их, пока область не перезаписана.

1. Войдите в Windows 10 под учётной записью с правами администратора.

#### 2. Основной этап

##### **Задание 1: Создание и форматирование разделов**

**1.1. Создание VHD через оснастку «Управление дисками»**  
- Нажмите `Win + R`, введите `diskmgmt.msc` и нажмите Enter.  
- В меню выберите **Действие** → **Создать виртуальный жесткий диск**.  
- Укажите расположение файла VHD (например, `C:\lab_vhd.vhd`), размер **10 ГБ**, тип **Динамически расширяемый**. Нажмите **ОК**.  
- В окне Управления дисками появится новый диск (скорее всего, **Диск 1**) с пометкой «Не инициализирован».  
- Правой кнопкой мыши (ПКМ) по области «Не инициализирован» → **Инициализировать диск**. Выберите **MBR** (или GPT, для лабораторной роли не принципиально).  
- Сделайте скриншот окна Управления дисками с инициализированным диском.

**1.2. Создание трёх разделов и форматирование**  
- ПКМ по нераспределённому пространству → **Создать простой том**.  
- Мастер создания томов: укажите размер первого раздела **3 ГБ** (или 3072 МБ).  
- Назначьте букву диска (например, **N** для NTFS).  
- На шаге форматирования выберите:  
  - **Файловая система:** NTFS  
  - **Метка тома:** NTFS_vol  
  - **Размер единицы размещения:** по умолчанию  
  - Установите флажок **Быстрое форматирование**.  
- Повторите для второго раздела (размер 3 ГБ, файловая система **FAT32**, метка `FAT32_vol`, буква `F`).  
- Повторите для третьего раздела (оставшееся пространство, примерно 4 ГБ, файловая система **exFAT**, метка `exFAT_vol`, буква `E`).  
- Сделайте скриншот, на котором видны три раздела с разными файловыми системами.

**1.3. Создание файлов и папок на разделах**  
- Откройте проводник и перейдите на диск N: (NTFS). Создайте папку `Documents` и текстовый файл `test.txt`.  
- На диске F: (FAT32) создайте папку `Data`.  
- На диске E: (exFAT) создайте папку `Media`.  
- Сделайте скриншот содержимого дисков.

**1.4. Работа с VHD через командную строку (diskpart)**  
- Закройте оснастку `diskmgmt.msc`.  
- Откройте командную строку от имени администратора (`Win + R` → `cmd` → `Ctrl+Shift+Enter`).  
- Выполните команды:
  ```cmd
  diskpart
  select vdisk file="C:\lab_vhd.vhd"
  attach vdisk
  list volume
  ```
- Убедитесь, что смонтированный VHD отображается и его разделы видны (буквы N, F, E). Сделайте скриншот.
- Выберите том с FAT32 (буква F) и выполните быстрое форматирование:
  ```cmd
  select volume F
  format fs=fat32 quick label="FAT32_new"
  ```
- После форматирования проверьте, что файлы на этом томе удалились. Сделайте скриншот.
- Отключите VHD:
  ```cmd
  detach vdisk
  exit
  ```

**Вопросы для отчёта (Задание 1):**
- Чем отличается процесс форматирования в FAT32 для дисков более 32 ГБ? (стандартные средства Windows не позволяют форматировать в FAT32 разделы >32 ГБ, но это ограничение искусственное, обходится сторонними утилитами)
- Почему exFAT рекомендуется для USB-накопителей? (нет ограничения 4 ГБ на файл, поддерживается на всех современных ОС, имеет меньшие накладные расходы, чем NTFS, и не требует журналирования)

##### **Задание 2: Анализ свойств файловых систем**

**2.1. Настройка прав доступа (только для NTFS)**  
- На диске N: (NTFS) создайте папку `Secret`.  
- ПКМ по папке → **Свойства** → вкладка **Безопасность** → **Изменить**.  
- Выберите группу **Пользователи** (Users) и поставьте галочку **Запретить** для разрешения **Полный доступ**. Нажмите **ОК**, **ОК**.  
- Попробуйте открыть папку `Secret` из-под текущего пользователя (доступ должен быть запрещён). Сделайте скриншот сообщения об отказе в доступе.  
- Верните разрешения (разрешить полный доступ) или снимите запрет.

**2.2. Сжатие данных (только для NTFS)**  
- На диске N: создайте файл размером 100 МБ с помощью команды в командной строке (от администратора):
  ```cmd
  fsutil file createnew N:\bigfile.txt 104857600
  ```
- Откройте свойства диска N: (ПКМ по диску → Свойства). На вкладке **Общие** установите флажок **Сжать этот диск для экономии места**. Нажмите **Применить** → выберите применить ко всем подпапкам и файлам. Дождитесь окончания операции.  
- Откройте свойства файла `N:\bigfile.txt` (ПКМ → Свойства). Сравните **Размер** (100 МБ) и **Размер на диске** (должен быть меньше). Сделайте скриншот свойств файла.  
- Отключите сжатие диска (снимите флажок).

**2.3. Проверка ограничений FAT32**  
- Перейдите на диск F: (FAT32). Попробуйте скопировать на него файл размером более 4 ГБ (например, `bigfile.txt` с диска N: не подойдёт, так как он 100 МБ). Создайте или найдите реальный файл >4 ГБ (например, образ DVD или воспользуйтесь `fsutil` для создания 5 ГБ файла на другом разделе NTFS).  
- Если нет большого файла, создайте его на диске N: командой:
  ```cmd
  fsutil file createnew N:\hugefile.dat 5368709120
  ```
- Скопируйте `N:\hugefile.dat` на диск F:. Запишите сообщение об ошибке (слишком большой файл для целевой файловой системы). Сделайте скриншот ошибки.  
- Удалите созданные файлы.

**Вопросы для отчёта (Задание 2):**

- Почему файл размером 5 ГБ нельзя записать на FAT32? (максимальный размер файла в FAT32 — 4 ГБ, из-за использования 32-битных указателей на кластеры)

##### **Задание 3: Диагностика и восстановление файловой системы**

**3.1. Использование `chkdsk`**  
- Убедитесь, что VHD подключён (см. шаг 1.4).  
- Откройте командную строку от администратора.  
- Выполните проверку тома с ошибками (например, диска F:):
  ```cmd
  chkdsk F: /f /r
  ```
- Дождитесь завершения. Сделайте скриншот вывода команды.  
- Расшифруйте параметры: `/f` — исправление ошибок на диске, `/r` — поиск повреждённых секторов и восстановление читаемой информации (неявно включает `/f`).

**3.2. Восстановление данных с помощью Recuva**  
- На диске N: (NTFS) создайте текстовый файл `secret.txt` с любым содержимым.  
- Удалите этот файл (в Корзину, затем очистите Корзину).  
- Запустите **Recuva** (portable).  
- В мастере выберите тип файлов «Документы», укажите место поиска «На диске N:\», нажмите **Сканировать**.  
- После сканирования найдите удалённый файл `secret.txt` (отметьте флажком). Нажмите **Восстановить** и выберите папку для восстановления (например, рабочий стол).  
- Сделайте скриншот окна Recuva с найденным файлом.  
- Убедитесь, что файл восстановлен.

**3.3. Завершение работы с VHD**  
- Закройте Recuva.  
- В командной строке от администратора выполните:
  ```cmd

  select vdisk file="C:\lab_vhd.vhd"

  exit
  ```
