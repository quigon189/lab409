### Тема
Мониторинг процессов и потоков в операционной системе Windows

### Цель работы
- **Теоретическая:** Изучить основные понятия: процесс, поток, дескриптор, идентификатор процесса (PID), приоритеты; понять принципы работы диспетчера задач, командных утилит и специализированного инструмента Process Explorer.
- **Практическая:** Приобрести навыки анализа процессов и потоков с использованием встроенных средств Windows (Диспетчер задач, командная строка) и сторонней утилиты Process Explorer, научиться определять ресурсоёмкие процессы, управлять ими и интерпретировать полученные данные.

### Задачи
1. Научиться запускать и настраивать Диспетчер задач для отображения необходимых параметров процессов.
2. Провести анализ процессов по загрузке CPU и памяти, выявить процессы с наибольшим потреблением ресурсов.
3. Изучить особенности процесса `svchost.exe` и его связь со службами.
4. Освоить работу с командными утилитами `tasklist` и `taskkill` для получения списка процессов и завершения их по PID или имени.
5. Познакомиться с утилитой Process Explorer от Sysinternals: изучить дерево процессов, потоки, стек вызовов, родительские процессы.
6. Провести сравнительный анализ инструментов мониторинга.

### Оборудование и программное обеспечение
- **Программное обеспечение:** Операционная система Windows (любая современная версия: 10, 11, Server 2016/2019/2022).
- **Дополнительное ПО:** Утилита Process Explorer (portable версия, предоставленная преподавателем или скачанная с официального сайта Microsoft Sysinternals).
- **Оборудование (виртуальное/реальное):** Персональный компьютер или виртуальная машина с Windows, доступ к учётной записи с правами администратора (для некоторых операций).

### Краткие теоретические сведения

**Процесс** — это экземпляр выполняемой программы, который имеет своё виртуальное адресное пространство, загруженный код, данные и ресурсы (открытые файлы, дескрипторы). Каждый процесс идентифицируется уникальным номером — **PID (Process Identifier)**.

**Поток (thread)** — минимальная единица выполнения, принадлежащая процессу. Один процесс может содержать несколько потоков, которые совместно используют память и ресурсы процесса, но выполняются независимо. Многопоточность позволяет эффективно использовать многоядерные процессоры и выполнять несколько задач одновременно.

**Дескриптор (handle)** — абстрактная ссылка на системный ресурс (файл, раздел реестра, сокет, процесс и т.д.), которую использует процесс для доступа к этому ресурсу. У каждого процесса есть таблица дескрипторов.

**Мониторинг процессов** необходим для выявления программ, потребляющих чрезмерные ресурсы, обнаружения сбоев, зависаний, а также для поиска подозрительной активности (вредоносное ПО).

Основные инструменты мониторинга в Windows:
- **Диспетчер задач** — встроенное приложение, предоставляющее базовую информацию о процессах, производительности, службах, автозагрузке.
- **Командная строка (cmd) и PowerShell** — позволяют получать детальные списки процессов с помощью команд `tasklist`, `wmic`, `Get-Process` и управлять ими через `taskkill`.
- **Process Explorer** — мощная утилита от Sysinternals, которая показывает дерево процессов, подробную информацию о каждом процессе (открытые дескрипторы, загруженные DLL, потоки, стек вызовов, сетевые соединения). Цветовая маркировка помогает быстро идентифицировать типы процессов (службы, приложения администратора, .NET-приложения).

### Порядок выполнения работы

> **Важно:** По каждому пункту работы необходимо сделать скриншот для последующего включения в отчёт.

#### 1. Подготовительный этап
1. Убедитесь, что вы вошли в систему с учётной записью, имеющей права администратора (для некоторых действий могут потребоваться повышенные привилегии).
2. Скопируйте portable-версию **Process Explorer** из предоставленного источника (сетевая папка, флешка) на локальный диск, например, `D:\ProcessExplorer`.
3. Подготовьте текстовый редактор для записи результатов и таблиц.

#### 2. Основной этап

##### **Задание 1: Базовый анализ через Диспетчер задач**

**1. Запуск Диспетчера задач**  
- Нажмите сочетание клавиш **Ctrl + Shift + Esc** (или **Ctrl + Alt + Delete** → выберите «Диспетчер задач»).  

**2. Настройка отображения столбцов**  
- Перейдите на вкладку **Процессы**.  
- В верхнем меню нажмите **«Вид»** → **«Выбрать столбцы»** и отметьте:  
  - **PID (Идентификатор процесса)**  
  - **ЦП (Загрузка процессора)**  
  - **Память (Потребление оперативной памяти)**  
  - **Описание**  

**3. Анализ загрузки CPU**  
- Нажмите на заголовок столбца **«ЦП»** для сортировки по убыванию загрузки процессора.  
- Сделайте скриншот.  
- Запишите в отчёт названия трёх процессов с наибольшей загрузкой CPU и кратко объясните, почему они могут потреблять много ресурсов (например, браузер при просмотре видео, антивирус, системные процессы).

**4. Анализ процесса svchost.exe**  
- Найдите все экземпляры процесса **svchost.exe** (можно использовать поиск или сортировку по имени).  
- Сделайте скриншот.  
- Перейдите на вкладку **Службы** и отсортируйте по столбцу **PID**.  
- Сделайте скриншот вкладки «Службы».  
- Ответьте на вопросы:  
  - Сколько экземпляров **svchost.exe** запущено?  
  - Почему их несколько?  
  - Как определить, какие службы связаны с каждым экземпляром? (подсказка: на вкладке «Службы» можно увидеть PID, сопоставить с PID из списка процессов).

**5. Определение процесса с наибольшим потреблением памяти**  
- Нажмите на заголовок столбца **«Память»**, чтобы отсортировать процессы по убыванию использования ОЗУ.  
- Сделайте скриншот.  
- Запишите:  
  - Название процесса, который использует больше всего памяти.  
  - Сколько мегабайт (МБ) он занимает.  
  - Предположите, почему этот процесс требует столько памяти (например, браузер с множеством вкладок, виртуальная машина, редактор изображений).

**6. Анализ вкладки «Быстродействие»**  
- Перейдите на вкладку **Быстродействие**.  
- Сделайте скриншот.  
- Зафиксируйте текущие значения:  
  - **Потоков**  
  - **Процессов**  
  - **Дескрипторов**  
- Объясните, что означают эти термины (можно описать своими словами).

**7. Наблюдение за динамикой при запуске браузера**  
- Закройте все приложения, кроме Диспетчера задач.  
- Запустите браузер (например, Google Chrome).  
- Вернитесь в Диспетчер задач, сделайте скриншот вкладки «Быстродействие» после запуска.  
- Заполните таблицу сравнения (пример ниже) и рассчитайте изменения.  

**Таблица для отчёта:**

| Показатель              | До запуска браузера | После запуска браузера | Изменение |
| ----------------------- | ------------------- | ---------------------- | --------- |
| Количество процессов    |                     |                        |           |
| Количество потоков      |                     |                        |           |
| Количество дескрипторов |                     |                        |           |

- Проанализируйте, на сколько увеличилось количество процессов, потоков и дескрипторов. Сделайте вывод.

##### **Задание 2: Углублённый анализ процессов через командную строку**

**1. Запуск командной строки**  
- Нажмите **Win + R**, введите `cmd.exe` и нажмите **Enter**.  

**2. Изучение команды `tasklist`**  
- Введите команду `tasklist /?` для просмотра справки.  
- Сделайте скриншот справки. Обратите внимание на доступные ключи (например, `/FI` для фильтрации, `/SVC` для отображения служб в процессах).

**3. Получение списка процессов**  
- Выполните команду `tasklist`.  
- Сделайте скриншот результата. Обратите внимание на колонки: **Image Name**, **PID**, **Session Name**, **Mem Usage**.

**4. Фильтрация вывода**  
- Выполните команду `tasklist | findstr "svchost.exe"` для поиска всех экземпляров `svchost.exe`.  
- Сделайте скриншот.  
- Запишите, сколько экземпляров найдено, и перечислите их PID.

**5. Завершение процесса с помощью `taskkill`**  
- Запустите **Блокнот** (например, командой `notepad.exe` в командной строке).  
- Определите его PID: `tasklist | findstr "notepad.exe"`.  
- Завершите процесс командой `taskkill /PID <PID> /F`.  
- Сделайте скриншот выполнения команды. Убедитесь, что окно Блокнота закрылось.

**6. Сравнение списка процессов до и после создания нагрузки**  
- Сохраните исходный список процессов в файл:  
  ```cmd
  tasklist > processes_before.txt
  ```
- Запустите три окна Блокнота и браузер.  
- Сохраните новый список:  
  ```cmd
  tasklist > processes_after.txt
  ```
- Сравните файлы с помощью команды:  
  ```cmd
  fc processes_before.txt processes_after.txt
  ```
- Сделайте скриншот результата сравнения.  
- Запишите, какие новые процессы появились (например, `notepad.exe` трижды, процесс браузера). Определите, сколько памяти потребляет браузер (найдите его в выводе `tasklist`).

##### **Задание 3: Детальный анализ процессов и потоков в Process Explorer**

**1. Запуск Process Explorer**  
- Перейдите в папку с утилитой и запустите `procexp.exe` (или `procexp64.exe` для 64-разрядной системы).  
- Если программа запускается впервые, согласитесь с лицензией.  

**2. Знакомство с интерфейсом**  
- Обратите внимание на цветовую маркировку:  
  - **Розовый** — процессы, запущенные от имени администратора.  
  - **Синий** — службы Windows.  
  - **Жёлтый** — .NET-приложения.  
- Сделайте скриншот общего окна.

**3. Анализ процесса svchost.exe в Process Explorer**  
- Найдите несколько экземпляров `svchost.exe`.  
- Раскройте дерево одного из них (нажмите на `+` слева).  
- Наведите курсор на процесс `svchost.exe` — появится всплывающая подсказка со списком служб, которые он содержит.  
- Сделайте скриншот с раскрытым деревом и всплывающей подсказкой.  
- Запишите:  
  - Сколько экземпляров `svchost.exe` запущено?  
  - Какие службы связаны с выбранным экземпляром?  
  - Почему Windows группирует несколько служб в одном процессе `svchost.exe`? (экономия памяти, облегчение управления).

**4. Изучение потоков процесса explorer.exe**  
- Найдите процесс `explorer.exe` (Проводник Windows).  
- Дважды кликните по нему, чтобы открыть окно свойств.  
- Перейдите на вкладку **Threads** (Потоки).  
- Сделайте скриншот вкладки с потоками.  
- Обратите внимание на столбцы: **TID** (идентификатор потока), **Start Address** (адрес начала), **Cycles Delta** (количество процессорных циклов).  
- Выберите любой поток и нажмите кнопку **Stack...** — откроется стек вызовов.  
- Сделайте скриншот стека.  
- Запишите, сколько потоков у процесса `explorer.exe`, и попробуйте определить, какой модуль (DLL) чаще всего встречается в стеке активного потока.

**5. Определение родительского процесса**  
- Запустите приложение **Калькулятор** (`calc.exe`).  
- В Process Explorer найдите процесс `calc.exe`.  
- Наведите на него курсор — во всплывающем окне будет указан родительский процесс.  
- Сделайте скриншот с подсказкой.  
- Запишите, какой процесс запустил Калькулятор (скорее всего, `explorer.exe`).  
- Объясните, почему некоторые процессы имеют родительский процесс `services.exe` или `wininit.exe` (системные службы запускаются этими процессами).

**6. Анализ потоков при высокой нагрузке**  
- Запустите ресурсоёмкое приложение (например, браузер с видео или тяжёлым сайтом).  
- Найдите его процесс в Process Explorer.  
- Откройте свойства процесса и перейдите на вкладку **Threads**.  
- Отсортируйте потоки по столбцу **CPU Usage** (кликнув по заголовку).  
- Сделайте скриншот.  
- Запишите, сколько потоков активно используют CPU, и какие библиотеки (DLL) связаны с этими потоками (можно увидеть в колонке **Start Address** или через стек).
