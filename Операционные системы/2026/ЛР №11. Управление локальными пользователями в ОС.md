### Тема
Управление локальными пользователями и группами, настройка прав доступа к файлам и каталогам в операционных системах Windows и Linux.

### Цель работы
- **Теоретическая:** Изучить механизмы создания и управления локальными пользователями и группами, принципы разграничения доступа к файловой системе в Windows (разрешения NTFS) и Linux (права доступа на чтение, запись, выполнение; владелец; группа).
- **Практическая:** Приобрести навыки создания пользователей и групп через графические и командные инструменты, настройки прав доступа к каталогам и файлам, проверки действия разрешений, удаления пользователей и их домашних каталогов.

### Задачи
1. **В Windows:**
   - Создать локальную группу и локального пользователя с помощью оснастки «Локальные пользователи и группы».
   - Создать на диске C: каталоги и текстовые файлы.
   - Настроить различные разрешения NTFS для созданного пользователя.
   - Проверить доступ (чтение, запись, запрет).
   - Удалить пользователя.
2. **В Linux (Manjaro):**
   - Создать пользователя с помощью `adduser`.
   - Настроить права на использование `sudo` через создание файла в `/etc/sudoers.d/`.
   - Создать каталоги и файлы, изменить владельца и права доступа (`chown`, `chmod`).
   - Проверить работу sudo и доступ к файлам/каталогам от имени нового пользователя.
   - Удалить пользователя и его домашний каталог (`deluser --remove-home`).

### Оборудование и программное обеспечение
- **Windows:** Операционная система Windows 10 (или 11) с правами администратора.
- **Linux:** Дистрибутив Manjaro (или любой Arch-совместимый) с правами `sudo`.
- **Оборудование:** Персональный компьютер или виртуальная машина с установленными ОС.

### Краткие теоретические сведения

**Управление локальными пользователями в Windows** осуществляется через оснастку `lusrmgr.msc` (Локальные пользователи и группы) – доступна в редакциях Pro и Enterprise. Для обычных пользователей также можно использовать командную строку (`net user`, `net localgroup`). Пользователи и группы хранятся локально в базе SAM. Разрешения NTFS на файлы и папки задаются через вкладку «Безопасность». Можно разрешать или запрещать чтение, запись, изменение, полный доступ для отдельных пользователей или групп. Запреты имеют приоритет над разрешениями.

**Управление пользователями в Linux** осуществляется командами `adduser` (интерактивный) или `useradd`. Параметры пользователя хранятся в `/etc/passwd`, пароли (хэши) – в `/etc/shadow`. Группы – в `/etc/group`. Права доступа к файлам делятся на три категории: владелец (user), группа (group), остальные (others). Изменить владельца можно `chown`, права – `chmod` (символьный или восьмеричный режим). Для предоставления прав на выполнение команд от root используется механизм `sudo`. Настройки sudo хранятся в `/etc/sudoers` и включаемых файлах в `/etc/sudoers.d/`.

## Часть 1. Windows 10

#### 1. Подготовительный этап (Windows)
1. Войдите в Windows под учётной записью **администратора**.

#### 2. Основной этап (Windows)

##### **Задание 1.1: Создание группы и пользователя**
1. Откройте оснастку **Локальные пользователи и группы**:
   - Нажмите `Win + R`, введите `lusrmgr.msc`, нажмите Enter.
2. Создайте новую группу:
   - ПКМ по папке **Группы** → **Создать группу**.
   - **Имя группы:** `TestGroup_<ваша_фамилия>`, например `TestGroup_Ivanov`.
   - **Описание:** `Группа для лабораторной работы`.
   - Нажмите **Создать**, затем **Закрыть**.
3. Создайте нового локального пользователя:
   - ПКМ по папке **Пользователи** → **Новый пользователь**.
   - **Имя пользователя:** `TestUser_<ваша_фамилия>`, например `TestUser_Ivanov`.
   - **Полное имя:** `Тестовый пользователь`.
   - **Пароль:** `P@ssw0rd`.
   - Снимите флажок **Требовать смену пароля при следующем входе**.
   - Установите флажок **Пароль не ограничен сроком действия**.
   - Нажмите **Создать**, затем **Закрыть**.
4. Добавьте пользователя в созданную группу:
   - Дважды кликните по группе `TestGroup_Ivanov` → вкладка **Члены** → **Добавить**.
   - Введите имя пользователя `TestUser_Ivanov`, нажмите **Проверить имена** → **ОК**.
5. Сделайте скриншот окна с созданной группой и пользователем (оснастка lusrmgr.msc).

##### **Задание 1.2: Создание каталогов и файлов на диске C:**
1. Откройте Проводник, перейдите на диск **C:\\**.
2. Создайте каталог `C:\Lab_Users`:
   - ПКМ → **Создать** → **Папку**, назовите `Lab_Users`.
3. Внутри `Lab_Users` создайте:
   - Каталог `ReadOnly`
   - Каталог `ReadWrite`
   - Текстовый файл `common.txt` (любое содержимое)
4. Сделайте скриншот содержимого `C:\Lab_Users`.

##### **Задание 1.3: Настройка разрешений NTFS**
1. **Настройка папки `ReadOnly`:**
   - ПКМ по папке `ReadOnly` → **Свойства** → вкладка **Безопасность**.
   - Нажмите **Изменить** → **Добавить** → введите `TestUser_Ivanov` → **ОК**.
   - Выделите пользователя `TestUser_Ivanov` в списке.
   - Разрешите: **Чтение и выполнение**, **Чтение**.
   - Запретите: **Запись**.
   - Нажмите **ОК**, затем **ОК**.
2. **Настройка папки `ReadWrite`:**
   - ПКМ по папке `ReadWrite` → **Свойства** → **Безопасность** → **Изменить** → **Добавить** → `TestUser_Ivanov`.
   - Разрешите: **Чтение и выполнение**, **Чтение**, **Запись**.
   - Нажмите **ОК**.
3. **Настройка файла `common.txt`:**
   - ПКМ по `common.txt` → **Свойства** → **Безопасность** → **Изменить** → **Добавить** → `TestUser_Ivanov`.
   - Разрешите: **Чтение**.
   - Нажмите **ОК**.
4. Сделайте скриншот окон настроек безопасности для каждой папки и файла.

##### **Задание 1.4: Проверка доступа от имени созданного пользователя**
1. Выйдите из текущего сеанса администратора (Пуск → значок пользователя → Выйти).
2. Войдите как `TestUser_Ivanov` с паролем `P@ssw0rd`.
3. Откройте Проводник и перейдите в `C:\Lab_Users`.
   - Попробуйте открыть папку `ReadOnly` → создать в ней файл (должна быть ошибка записи).
   - Попробуйте открыть папку `ReadWrite` → создать там текстовый файл (должно получиться).
   - Попробуйте открыть файл `common.txt` (чтение должно быть разрешено), попробуйте его изменить и сохранить (должна быть ошибка).
4. Сделайте скриншоты сообщений об ошибках и успешных операций.
5. Выйдите из сеанса `TestUser_Ivanov`, вернитесь под администратором.

##### **Задание 1.5: Удаление пользователя**
1. Откройте `lusrmgr.msc`.
2. Удалите пользователя `TestUser_Ivanov` (ПКМ → Удалить → Да).
3. Удалите группу `TestGroup_Ivanov`.
4. Сделайте скриншот, что пользователя и группы больше нет.
5. Удалите каталог `C:\Lab_Users`.

## Часть 2. Linux (Manjaro)

#### 1. Подготовительный этап (Linux)
1. Запустите Manjaro, откройте терминал.
#### 2. Основной этап (Linux)

##### **Задание 2.1: Создание пользователя с помощью `adduser`**
1. Создайте нового пользователя (интерактивно):
   ```bash
   sudo adduser testuser_<ваша_фамилия>
   ```
   Например: `sudo adduser testuser_ivanov`
2. Следуйте подсказкам:
   - **Пароль:** `P@ssw0rd` (повторить).
   - Остальные поля (Full Name, Room Number и т.д.) можно пропустить (просто нажимайте Enter).
3. Проверьте, что пользователь создан:
   ```bash
   id testuser_ivanov
   ```
   Сделайте скриншот вывода.

##### **Задание 2.2: Настройка прав на использование sudo**
1. Создайте файл в каталоге `/etc/sudoers.d/`:
   ```bash
   sudo visudo -f /etc/sudoers.d/testuser_ivanov
   ```
   (Используйте `visudo`, чтобы избежать синтаксических ошибок).
2. Разрешить пользователю выполнять только:
	- `systemctl status` (просмотр статуса служб),
	- `cat` (просмотр содержимого файлов),
	- `ls` в определённых каталогах.

3. Добавьте в файл строку:
   ```text
   testuser_ivanov ALL=(ALL:ALL) /usr/bin/systemctl status *, /bin/cat, /usr/bin/ls 
   ```
   Сохраните и закройте (в nano: `Ctrl+O`, `Enter`, `Ctrl+X`) (в vi: `Esc`, `:qw`).
   
4. Проверьте синтаксис:
   ```bash
   sudo visudo -c
   ```
   Сделайте скриншот успешной проверки.

##### **Задание 2.3: Создание каталогов и файлов от имени текущего пользователя**
1. Создайте рабочую директорию:
   ```bash
   mkdir ~/lab_users
   cd ~/lab_users
   ```
2. Создайте каталоги и файлы:
   ```bash
   mkdir readonly readwrite
   touch common.txt
   echo "Общий файл" > common.txt
   echo "Тест" > readwrite/write.txt
   ```
3. Просмотрите структуру:
   ```bash
   ls -la
   ```
   Сделайте скриншот.

##### **Задание 2.4: Изменение владельца и прав доступа**
1. Измените владельца всех созданных объектов на нового пользователя (пока от root):
   ```bash
   sudo chown -R testuser_ivanov:testuser_ivanov ~/lab_users
   ```
2. Настройте права:
   - Для каталога `readonly` – только чтение для владельца, без записи:
     ```bash
     chmod 555 readonly   # r-xr-xr-x – нельзя писать
     ```
   - Для каталога `readwrite` – полные права для владельца:
     ```bash
     chmod 755 readwrite  # rwxr-xr-x
     ```
   - Для файла `common.txt` – только чтение для всех:
     ```bash
     chmod 644 common.txt # rw-r--r--
     ```
3. Проверьте права:
   ```bash
   ls -la
   ```
   Сделайте скриншот.

##### **Задание 2.5: Проверка работы sudo и доступа к файлам от нового пользователя**
1. Переключитесь на нового пользователя:
   ```bash
   sudo -i -u testuser_ivanov
   ```
   (или `su - testuser_ivanov`, если знаете пароль).
2. Проверьте работу `sudo`:
   ```bash
   sudo cat /etc/passwd
   ```
3. Перейдите в каталог `/home/ваш_старый_пользователь/lab_users`:
  ```bash
  cd /home/user/lab_users
  ``` 
4. Проверьте доступ:
   - Попробуйте создать файл в `readonly`:
     ```bash
     touch readonly/test.txt
     ```
     Должна быть ошибка `Permission denied`. Сделайте скриншот.
   - Попробуйте создать файл в `readwrite`:
     ```bash
     touch readwrite/success.txt
     ```
     Должно успешно. Сделайте скриншот.
   - Попробуйте изменить `common.txt`:
     ```bash
     echo "new" >> common.txt
     ```
     Должна быть ошибка (только чтение). Сделайте скриншот.
5. Выйдите из сеанса нового пользователя (`exit`).

##### **Задание 2.6: Удаление пользователя и его домашнего каталога**
1. Убедитесь, что пользователь не залогинен (можно проверить `ps -u testuser_ivanov`).
2. Удалите пользователя вместе с его домашним каталогом и почтовым ящиком:
   ```bash
   sudo deluser --remove-home testuser_ivanov
   ```
3. Удалите файл sudoers:
   ```bash
   sudo rm /etc/sudoers.d/testuser_ivanov
   ```
4. Проверьте, что пользователь удалён:
   ```bash
   id testuser_ivanov
   ```
   Должна быть ошибка «no such user». Сделайте скриншот.
