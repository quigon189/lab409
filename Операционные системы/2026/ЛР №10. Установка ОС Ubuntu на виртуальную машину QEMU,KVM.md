### Тема
Установка гостевой ОС Ubuntu Server на виртуальную машину, работающую под управлением гипервизора QEMU/KVM, на хосте с Manjaro Linux, с использованием графического интерфейса Virt-Manager.

### Цель работы
- **Теоретическая:** Изучить архитектуру виртуализации на базе KVM/QEMU, принципы работы мостового сетевого подключения, форматы виртуальных дисков (QCOW2), а также инструменты управления виртуальными машинами (libvirt, Virt-Manager).
- **Практическая:** Приобрести навыки установки и настройки KVM/QEMU в дистрибутиве Manjaro, создания виртуальной машины с Ubuntu Server, настройки сетевого моста для доступа из внешней сети, управления ВМ через Virt-Manager и командную строку.

### Задачи
1. Установить пакеты QEMU, KVM, libvirt, Virt-Manager и мостовые утилиты в Manjaro.
2. Настроить пользовательские группы для доступа к KVM и libvirt.
3. Создать и настроить мостовой сетевой интерфейс (bridge) для прямого доступа к сети хоста.
4. Запустить Virt-Manager и создать новую виртуальную машину с параметрами: 1024 МБ ОЗУ, 2 vCPU, диск 20 ГБ (QCOW2), сетевая карта на мосте br0.
5. Установить Ubuntu Server (22.04 или последнюю LTS) с русским языком, именем пользователя `qemu-<фамилия>`, паролем `P@ssw0rd`, включив сервер OpenSSH.
6. Проверить работоспособность: доступ по SSH, версия ОС, статус SSH-службы.
7. Освоить базовые команды управления ВМ через `virsh` (опционально).

### Оборудование и программное обеспечение
- **Программное обеспечение хоста:** Manjaro Linux (свежая версия, с ядром 5.15+), поддержка аппаратной виртуализации (Intel VT-x / AMD-V).
- **Программное обеспечение гостя:** ISO-образ Ubuntu Server (22.04 LTS или новее) – скачать с официального сайта.
- **Устанавливаемые пакеты:** `qemu-desktop`, `libvirt`, `virt-manager`, `bridge-utils`, `dnsmasq`, `edk2-ovmf` (для UEFI, опционально).
- **Оборудование:** Персональный компьютер с не менее 4 ГБ ОЗУ и 30 ГБ свободного места на диске.

### Краткие теоретические сведения

**Виртуализация на базе KVM/QEMU** – стандартное решение для Linux. KVM (Kernel-based Virtual Machine) является модулем ядра, превращающим Linux в гипервизор 1-го типа. QEMU (Quick Emulator) предоставляет эмуляцию устройств и работает в связке с KVM для достижения нативной производительности.

**Компоненты:**
- **KVM** – обеспечивает изоляцию и выполнение машинного кода гостя напрямую на процессоре.
- **QEMU** – эмулирует периферию (сетевые карты, диски, графику).
- **libvirt** – набор библиотек и демон `libvirtd`, предоставляющий унифицированный API для управления ВМ.
- **Virt-Manager** – графическая программа (GTK) для управления виртуальными машинами через libvirt.
- **Мостовой сетевой мост (bridge)** – объединяет физический сетевой интерфейс хоста с виртуальными интерфейсами гостей, позволяя ВМ работать как полноценные узлы в локальной сети (получать IP от DHCP роутера, быть доступными по сети).

**Форматы дисков:**
- **QCOW2 (QEMU Copy-On-Write 2)** – родной формат QEMU с поддержкой снапшотов, сжатия, тонкого выделения места.
- **RAW** – побитовый образ, быстрее, но без дополнительных функций.

**Установка в Manjaro** отличается от Debian/Ubuntu использованием `pacman` и некоторыми путями конфигурации.

### Порядок выполнения работы

> **Важно:** По каждому пункту работы необходимо сделать скриншот для последующего включения в отчёт.

#### 1. Подготовительный этап
1. Убедитесь, что процессор поддерживает аппаратную виртуализацию:
   ```bash
   grep -E "vmx|svm" /proc/cpuinfo
   ```
   Если вывод не пуст – поддержка есть.
2. Установите необходимые пакеты в Manjaro:
   ```bash
   sudo pacman -Syu
   sudo pacman -S qemu-desktop libvirt virt-manager bridge-utils dnsmasq edk2-ovmf
   ```
   - `qemu-desktop` – полная сборка QEMU с графической поддержкой.
   - `libvirt` – демон управления.
   - `virt-manager` – графический клиент.
   - `bridge-utils` – утилиты для настройки мостов.
   - `dnsmasq` – для NAT-сетей (может потребоваться).
   - `edk2-ovmf` – прошивка UEFI для ВМ (рекомендуется).
3. Включите и запустите демон libvirt:
   ```bash
   sudo systemctl enable libvirtd.service
   sudo systemctl start libvirtd.service
   ```
4. Добавьте своего пользователя в группы `libvirt` и `kvm`:
   ```bash
   sudo usermod -aG libvirt,kvm $USER
   ```
   Выйдите из системы и зайдите заново (или выполните `newgrp libvirt` для текущей сессии).

#### 2. Основной этап

##### **Задание 1: Настройка сетевого моста (bridge)**

1. Определите имя вашего физического сетевого интерфейса (обычно `enpXsY`, `eth0` или `wlan0` для Wi-Fi):
   ```bash
   ip a
   ```
   Запомните имя (например, `enp2s0`).
2. Создайте конфигурационный файл моста для NetworkManager (Manjaro по умолчанию использует NetworkManager). Альтернативно можно настроить через `systemd-networkd`. Самый простой способ для Virt-Manager – использовать режим моста, управляемый libvirt (NAT по умолчанию), но для прямого доступа к физической сети мы создадим мост вручную.

   **Способ через NetworkManager (рекомендуется):**
   - Откройте терминал и выполните:
     ```bash
     nm-connection-editor
     ```
   - Нажмите «Добавить» → выберите «Мост» → создайте мост с именем `br0`.
   - Во вкладке «Мост» добавьте существующий физический интерфейс (например, `enp2s0`).
   - Во вкладке «IPv4» выберите метод «Автоматически (DHCP)».
   - Примените настройки. Удалите или отключите исходное подключение физического интерфейса.
   - **Альтернативно через командную строку**:
     ```bash
     sudo nmcli con add type bridge ifname br0
     sudo nmcli con add type bridge-slave ifname enp2s0 master br0
     sudo nmcli con up br0
     ```

   **Способ через systemd-networkd (если не используете NetworkManager):**
   - Создайте файл `/etc/systemd/network/bridge.netdev`:
     ```
     [NetDev]
     Name=br0
     Kind=bridge
     ```
   - Файл `/etc/systemd/network/enp2s0.network`:
     ```
     [Match]
     Name=enp2s0
     [Network]
     Bridge=br0
     ```
   - Файл `/etc/systemd/network/br0.network`:
     ```
     [Match]
     Name=br0
     [Network]
     DHCP=yes
     ```
   - Запустите `systemd-networkd`.

3. Проверьте, что мост получил IP-адрес:
   ```bash
   ip a show br0
   ```
   Сделайте скриншот вывода.

##### **Задание 2: Создание виртуальной машины через Virt-Manager**

1. Запустите Virt-Manager:
   ```bash
   virt-manager
   ```
   (Если не запускается от обычного пользователя, проверьте группы.)
2. В окне Virt-Manager нажмите **Создать новую виртуальную машину** (иконка светящегося монитора).
3. Выберите **Локальный установочный носитель (ISO-образ)** → **Вперёд**.
4. Нажмите **Выбрать образ** и укажите путь к скачанному ISO-файлу Ubuntu Server. Система автоматически определит тип ОС (Linux, Ubuntu 22.04). Нажмите **Вперёд**.
5. Настройте ресурсы:
   - **Память (RAM):** 1024 МБ
   - **Количество процессоров (CPU):** 2
   - **Вперёд**.
6. Настройка диска:
   - Выберите **Создать образ диска**.
   - **Размер:** 20 ГБ.
   - **Формат хранилища:** QCOW2 (по умолчанию).
   - **Вперёд**.
7. Имя виртуальной машины: `Ubuntu-VM-<ваша_фамилия>` (например, `Ubuntu-VM-Ivanov`).
   - **Сеть:** выберите ранее созданный мост `br0`. Если мост не отображается, можно указать «Мостовое устройство» и вписать `br0` вручную.
8. Нажмите **Готово**. Виртуальная машина запустится и начнёт загрузку с ISO.
9. Сделайте скриншот окна настроек ВМ (ресурсы, диск, сеть) и окна Virt-Manager с запущенной ВМ.

##### **Задание 3: Установка Ubuntu Server**

1. В графической консоли ВМ (откроется автоматически) следуйте шагам установщика Ubuntu Server:
   - **Язык:** Russian (или English, по желанию).
   - **Клавиатура:** Russian.
   - **Тип установки:** Ubuntu Server.
   - **Сеть:** автоматически через DHCP – проверьте, что интерфейс получил IP (обычно ens3). Продолжить.
   - **Прокси:** оставить пустым.
   - **Зеркало:** оставить стандартное.
   - **Метод установки:** Guided – use entire disk (авторазметка).
   - **Имя пользователя:** `qemu-<ваша_фамилия>` (например, `qemu-ivanov`).
   - **Имя компьютера:** `ubuntu-vm-<фамилия>`.
   - **Пароль:** `P@ssw0rd` (повторить).
   - **Установка OpenSSH server:** отметьте пробелом, затем Enter.
   - **Выбор дополнительных пакетов:** ничего не выбирать.
   - Дождаться завершения установки, затем **Reboot**.
2. После перезагрузки войдите в систему (логин/пароль).
3. Сделайте скриншот окна входа или приглашения командной строки.

##### **Задание 4: Проверка работы**

1. **Проверка IP-адреса** внутри ВМ:
   ```bash
   ip a
   ```
   Запишите IP-адрес (например, `192.168.1.123`).
2. **Подключение по SSH с хоста:**
   Откройте новый терминал на хосте Manjaro и выполните:
   ```bash
   ssh qemu-ivanov@<IP_адрес_ВМ>
   ```
   Введите пароль `P@ssw0rd`. Если подключение успешно – сетевой мост работает.
   Сделайте скриншот успешного SSH-подключения.
3. **Проверка версии ОС:**
   ```bash
   lsb_release -a
   ```
   Сделайте скриншот.
4. **Проверка статуса SSH-службы:**
   ```bash
   sudo systemctl status ssh
   ```
   (Пароль sudo – тот же `P@ssw0rd`). Сделайте скриншот.

##### **Задание 5 (дополнительное): Управление ВМ через virsh**

1. На хосте выполните команды для просмотра списка ВМ:
   ```bash
   virsh list --all
   ```
2. Остановите ВМ:
   ```bash
   virsh shutdown Ubuntu-VM-Ivanov
   ```
   (замените имя на своё)
3. Запустите снова:
   ```bash
   virsh start Ubuntu-VM-Ivanov
   ```
4. Сделайте скриншот вывода `virsh list`.

### Контрольные вопросы
1. Чем отличается QEMU от KVM? (KVM – модуль ядра для ускорения, QEMU – эмулятор устройств; вместе дают полную виртуализацию)
2. Как добавить постоянное хранилище для ВМ? (создать дополнительный диск в формате QCOW2 через Virt-Manager или `qemu-img`, затем подключить к ВМ через `virsh attach-disk` или в настройках)
3. Как настроить проброс портов для доступа к веб-серверу в гостевой ОС? (при использовании NAT можно добавить правило `iptables` или через `virsh net-edit`; при мостовом режиме проброс не требуется, так как гость имеет собственный IP)
4. Зачем нужен мостовой режим сети? (чтобы виртуальная машина была полноценным узлом локальной сети, получала IP от роутера, была доступна по сети без дополнительных настроек)
