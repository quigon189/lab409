### Краткие теоретические сведения

**Терминальные файловые менеджеры (CLI)** — программы для управления файлами через командную строку. Они незаменимы при работе с серверами (без графического интерфейса), в условиях ограниченных ресурсов или для автоматизации задач. Каждый менеджер имеет свой стиль и набор возможностей.

**Midnight Commander (MC)** — классический двухпанельный менеджер, появившийся в 1994 году. Особенности: традиционный интерфейс, поддержка горячих клавиш (F1–F10), встроенный текстовый редактор (mcedit), просмотрщик файлов, работа по SSH/FTP (через виртуальные файловые системы), поддержка мыши.

**Ranger** — текстовый менеджер с трёхпанельным интерфейсом (левая панель – текущий каталог, правая – содержимое выделенного файла/каталога, снизу – командная строка). Особенности: предпросмотр файлов (изображения, текст, видео через утилиты), навигация с помощью стрелок и vim-подобных клавиш, поддержка плагинов, возможность просматривать дерево каталогов.

**Vifm** — модальный файловый менеджер, использующий концепции редактора Vim. Двухпанельный интерфейс, управление через команды `:`, визуальный режим, поддержка конфигурационных файлов (vifmrc), интеграция с внешними утилитами (grep, tar, rsync). Подходит для пользователей, привыкших к Vim.

**nnn** — минималистичный и очень быстрый менеджер с двухпанельным интерфейсом. Особенности: низкое потребление ресурсов, работа с большими каталогами (миллионы файлов), поддержка плагинов через внешние скрипты, встроенные функции (desktop search, дисковый анализатор). Управление – через сочетания клавиш (без меню).

##### **Midnight Commander (MC)**

**Запуск:** `mc`

**Интерфейс:** две панели, внизу – подсказки по функциональным клавишам (F1–F10). Навигация: стрелки, Tab для переключения панелей, Enter для входа в каталог.

**Выполнение операций:**
1. **Создать каталог `mc_test`:**  
   - Нажмите `F7` (Создать каталог).  
   - Введите `mc_test`, нажмите Enter.
2. **Войти в каталог:** стрелками выделите `mc_test`, Enter.
3. **Создать файл `file.txt`:**  
   - Нажмите `Shift+F4` (создать/редактировать файл).  
   - Введите `file.txt`, Enter. В открывшемся редакторе (mcedit) можно сразу выйти `F10` (или написать текст и сохранить `F2`). Будет создан пустой файл.
4. **Создать подкаталог `backup`:** `F7` → `backup`.
5. **Скопировать `file.txt` в `backup`:**  
   - Выделите `file.txt` (стрелка вниз).  
   - Нажмите `F5` (Копировать).  
   - В поле «На какой файл/каталог» введите `backup/` (или `backup/file.txt`), Enter.
6. **Переименовать `file.txt` в `renamed.txt`:**  
   - Выделите `file.txt`, нажмите `F6` (Переименовать/переместить).  
   - Введите `renamed.txt`, Enter.
7. **Удалить каталог `mc_test` (находясь внутри него, нужно выйти на уровень выше):**  
   - Нажмите `F10` (выйти из редактора, если открыт).  
   - Перейдите в родительский каталог: `F10` (или `Esc`+`1` для выхода из текущей панели).  
   - Выделите `mc_test`, нажмите `F8` (Удалить). Подтвердите Enter.
8. **Поиск файла по имени (например, `renamed.txt`):**  
   - Нажмите `F9` для активации верхнего меню → стрелками выберите «Команда» → «Найти файл».  
   - Введите имя файла, укажите каталог поиска (можно начать с `/home/ваше_имя`), Enter.
9. **Работа с архивом:**  
   - Создайте временный каталог `~/temp` и войдите в него.  
   - Создайте файлы (например, `a.txt`, `b.txt`).  
   - Выделите их (`Insert` для множественного выбора) → нажмите `F9` → «Команда» → «Упаковать». Выберите `tar.gz`, укажите имя `test.tar.gz`.  
   - Для распаковки: выделите `test.tar.gz`, нажмите `F9` → «Команда» → «Распаковать».  
   - Можно также войти внутрь архива как в каталог (Enter на архиве, если в MC настроена поддержка).

##### **Ranger**

**Запуск:** `ranger`

**Интерфейс:** три панели (слева – список каталогов, справа – предпросмотр, внизу – командная строка). Навигация: стрелки (вверх/вниз – перемещение по файлам), `h` – вверх по дереву, `l` – войти в каталог/открыть файл, `q` – выход.

**Выполнение операций:**
1. **Создать каталог `ranger_test`:**  
   - Нажмите `mkdir` (начинается ввод команды внизу).  
   - Введите `mkdir ranger_test`, Enter.
2. **Войти в каталог:** стрелками выделить `ranger_test`, нажать `l`.
3. **Создать файл `file.txt`:**  
   - Нажмите `:touch file.txt`, Enter.
4. **Создать подкаталог `backup`:** `:mkdir backup`.
5. **Скопировать `file.txt` в `backup`:**  
   - Выделите `file.txt` (стрелками).  
   - Нажмите `yy` (скопировать в буфер).  
   - Перейдите в каталог `backup`: выделите `backup`, нажмите `l`.  
   - Нажмите `pp` (вставить скопированное).
6. **Переименовать `file.txt` в `renamed.txt`:**  
   - Вернитесь в родительский каталог (`h` или `..`).  
   - Выделите `file.txt`, нажмите `A` (rename) → введите `renamed.txt`, Enter.
7. **Удалить каталог `ranger_test`:**  
   - Выйдите из него (`h`).  
   - Выделите `ranger_test`, нажмите `dd` (удалить), подтвердите `y`.
8. **Поиск файла:**  
   - Нажмите `/`, введите имя файла, Enter. Ranger покажет результаты внизу.  
   - Для следующего совпадения – `n`, предыдущего – `N`.
9. **Работа с архивом:**  
   - Создайте каталог `~/temp`, войдите в него.  
   - Создайте файлы: `:touch a.txt b.txt`.  
   - Выделите их (`Space` для выбора).  
   - Нажмите `:shell tar -czf test.tar.gz %s` (где `%s` подставит выделенные имена).  
   - Распакуйте: выделите `test.tar.gz`, нажмите `:shell tar -xzf %f`.  
   - Также можно использовать предпросмотр – нажав Enter на архиве, если настроен плагин.

##### **Vifm**

**Запуск:** `vifm`

**Интерфейс:** две панели (похож на MC, но модальный). Команды вводятся через `:`. Выход – `:q`. Навигация: `j/k` или стрелки вверх/вниз, `h` – родительский каталог, `l` – войти. Режим командной строки – `;` или `:`. Режим визуального выделения – `v`.

**Выполнение операций:**
1. **Создать каталог `vifm_test`:**  
   - Нажмите `:mkdir vifm_test`, Enter.
2. **Войти в каталог:** выделить, нажать `l`.
3. **Создать файл `file.txt`:**  
   - Нажмите `:touch file.txt`, Enter.
4. **Создать подкаталог `backup`:** `:mkdir backup`.
5. **Скопировать `file.txt` в `backup`:**  
   - Выделите `file.txt`, нажмите `yy` (копировать).  
   - Перейдите в `backup` (`l`).  
   - Нажмите `pp` (вставить).
6. **Переименовать `file.txt` в `renamed.txt`:**  
   - Вернитесь в родительский каталог (`h`).  
   - Выделите `file.txt`, нажмите `cw` (change word – переименовать), введите `renamed.txt`, Enter.
7. **Удалить каталог `vifm_test`:**  
   - Выйдите из него (`h`).  
   - Выделите каталог, нажмите `dd` (удалить), подтвердите `y`.
8. **Поиск файла:**  
   - Нажмите `/`, введите имя файла, Enter.  
   - Для следующего совпадения – `n`.
9. **Работа с архивом:**  
   - В каталоге `~/temp` создайте файлы `:touch a.txt b.txt`.  
   - Выделите их (`v` или `V`).  
   - Нажмите `:` и введите `!tar -czf test.tar.gz %f` (где `%f` – выделенные файлы).  
   - Распаковка: выделите `test.tar.gz`, нажмите `:!tar -xzf %f`.

##### **nnn**

**Запуск:** `nnn`

**Интерфейс:** две панели (по умолчанию), но может быть одна. Навигация: стрелки, Enter – войти, `q` – выход. Справка: `?`. Команды выполняются через `!` или `;`. Основные сочетания: `n` – новый файл/каталог, `d` – удалить, `r` – переименовать, `y` – копировать, `p` – вставить.

**Выполнение операций:**
1. **Создать каталог `nnn_test`:**  
   - Нажмите `n` → появится запрос «Create new:». Введите `nnn_test`, нажмите Enter (по умолчанию создаётся каталог, для файла нужно было бы `-f`).
2. **Войти в каталог:** выделить, Enter.
3. **Создать файл `file.txt`:**  
   - Нажмите `n`, введите `file.txt` (будет создан пустой файл).
4. **Создать подкаталог `backup`:** `n` → `backup`.
5. **Скопировать `file.txt` в `backup`:**  
   - Выделите `file.txt`, нажмите `y` (копировать в буфер).  
   - Перейдите в `backup` (Enter).  
   - Нажмите `p` (вставить).  
   - Появится запрос «copy here?» – нажмите `y` или Enter.
6. **Переименовать `file.txt` в `renamed.txt`:**  
   - Вернитесь в родительский каталог (нажмите `-` или `..`).  
   - Выделите `file.txt`, нажмите `r`, введите `renamed.txt`, Enter.
7. **Удалить каталог `nnn_test`:**  
   - Выйдите из него (на уровень выше).  
   - Выделите каталог, нажмите `d`, подтвердите `y`.
8. **Поиск файла:**  
   - Нажмите `/` (или `F`), введите имя файла. nnn покажет совпадения.  
   - Для навигации по результатам используйте стрелки.
9. **Работа с архивом:**  
   - В каталоге `~/temp` создайте файлы `a.txt`, `b.txt` (с помощью `n`).  
   - Выделите их (пробел для множественного выбора, `Ctrl+A` – выделить все).  
   - Нажмите `!` (выполнить внешнюю команду). Введите `tar -czf test.tar.gz $nnn` (где `$nnn` – выделенные файлы).  
   - Распаковка: выделите `test.tar.gz`, нажмите `!` и введите `tar -xzf $nnn`.  
   - Примечание: в nnn нет встроенной работы с архивами, только через внешние команды.
