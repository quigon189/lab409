## Теоретические сведения

### Основные характеристики цифровых кодов

При передаче данных по кабельным системам (витая пара, коаксиальный кабель) цифровой сигнал должен удовлетворять ряду требований:
- **Отсутствие постоянной составляющей** (DC balance) — чтобы избежать смещения уровня и искажения на трансформаторах.
- **Самосинхронизация** — наличие в сигнале переходов, достаточных для выделения тактовой частоты приёмником.
- **Ограниченная полоса частот** — спектр сигнала не должен выходить за пределы, поддерживаемые кабелем.
- **Обнаружение ошибок** — некоторые коды позволяют выявлять ошибочные биты.

### Классификация методов кодирования

| Метод                                  | Принцип                                                                                    | Применение                                                                                               |
| -------------------------------------- | ------------------------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------------------- |
| **NRZ** (Non-Return to Zero)           | 1 — высокий уровень, 0 — низкий уровень                                                    | Последовательные интерфейсы (RS-232, USB 1.x), не используется в Ethernet из-за отсутствия синхронизации |
| **NRZI** (Non-Return to Zero Inverted) | Переход уровня при 1, отсутствие перехода при 0                                            | USB, Fast Ethernet (100BASE-TX) как часть кодирования 4B/5B                                              |
| **Manchester**                         | Переход в середине бита: 1 — высокий→низкий, 0 — низкий→высокий                            | Ethernet 10BASE-T. Гарантирует переход в каждом бите, требует вдвое большей полосы                       |
| **Дифференциальный Манчестер**         | Переход в начале бита при 0, отсутствие перехода в начале при 1; всегда переход в середине | Token Ring, старые сети                                                                                  |
| **MLT-3**                              | Три уровня, переход на соседний уровень при 1, отсутствие перехода при 0                   | Fast Ethernet 100BASE-TX (после 4B/5B)                                                                   |
| **4B/5B**                              | Блочное кодирование: 4 бита → 5 бит с балансом единиц/нулей                                | 100BASE-TX, FDDI                                                                                         |
| **8B/10B**                             | 8 бит → 10 бит, баланс единиц/нулей, запрещённые последовательности                        | Gigabit Ethernet (1000BASE-X), PCI Express, SATA                                                         |

### Основные понятия

- **Период бита** \(T\) — время, отводимое на передачу одного бита информации.
- **Скважность импульсов** — отношение периода повторения к длительности импульса.
- **Постоянная составляющая** — среднее значение сигнала за длительное время. Ненулевая постоянная составляющая приводит к насыщению трансформаторов и искажению сигнала.
- **Самосинхронизация** — способность приёмника выделять тактовую частоту непосредственно из принимаемого сигнала без отдельной тактовой линии.

### Задание 1. Временные диаграммы сигналов

**Исходные данные:** двоичная последовательность **10110010**.

Для этой последовательности необходимо построить временные диаграммы следующих кодов:
1. NRZ (полярный)
2. NRZI
3. Манчестер (бифазный)
4. MLT-3 (начальный уровень = 0, переходы: +1, 0, -1, 0, +1...)

#### Описание построения временной диаграммы

Временная диаграмма (осциллограмма) цифрового сигнала строится в прямоугольных координатах: **горизонтальная ось — время**, **вертикальная ось — уровень сигнала**.

**Общие правила:**
- Каждый бит последовательности отображается на интервале длительностью \(T\) (период бита). Границы битов следует отметить вертикальными пунктирными линиями.
- Для кодов с несколькими уровнями (например, MLT-3) вертикальную ось размечают соответствующими значениями (обычно -1, 0, +1).
- Начало диаграммы — момент времени \(t = 0\). Для определённости считаем, что перед первым битом линия находится в состоянии «0» (для NRZ, NRZI, Манчестера и дифф. Манчестера — нулевой уровень; для MLT-3 — уровень 0).
- Изменения уровня могут происходить только в моменты границ битов (для NRZ, NRZI) или в середине бита (для Манчестера и дифференциального Манчестера), или в моменты битов для MLT-3.

**Пошаговая инструкция для каждого кода:**

1. **NRZ (Non-Return to Zero)**
   - Уровень сигнала постоянен в течение всего бита.
   - «1» → высокий уровень (например, +1 усл. ед.).
   - «0» → низкий уровень (например, 0 усл. ед.).
   - Переходы происходят только на границах битов.

2. **NRZI (Non-Return to Zero Inverted)**
   - Начальный уровень (перед первым битом) принять за «0» (низкий).
   - При передаче «1» — уровень сигнала меняется на противоположный.
   - При передаче «0» — уровень остаётся прежним.
   - Переходы (если есть) — только на границах битов.

3. **Манчестер (бифазный)**
   - В каждом бите обязательно происходит переход в середине бита.
   - «1» кодируется переходом из высокого уровня в низкий (+1 → 0) в середине бита.
   - «0» кодируется переходом из низкого уровня в высокий (0 → +1) в середине бита.
   - На границах битов переходы могут быть (для непрерывности), но не обязательны.

4. **MLT-3 (Multi-Level Transition 3)**
   - Используются три уровня: +1, 0, -1. Переход между ними происходит последовательно: 0 → +1 → 0 → -1 → 0 → +1 → ...
   - При передаче бита «1»: уровень сигнала сдвигается к следующему значению в последовательности (не важно, вверх или вниз).
   - При передаче бита «0»: уровень сигнала остаётся неизменным.
   - Начальный уровень (перед первым битом) = 0.
   - Изменение уровня (если есть) происходит в начале бита (на границе) и удерживается весь бит.

**Результат выполнения задания:**
