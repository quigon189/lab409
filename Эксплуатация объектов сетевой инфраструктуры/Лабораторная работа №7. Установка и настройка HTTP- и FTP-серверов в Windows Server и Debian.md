## **Тема**
Установка и настройка веб-сервера (HTTP) и FTP-сервера в операционных системах Windows Server и Debian.

## **Цель работы**
- **Теоретическая:** Изучить принципы работы протоколов HTTP и FTP, архитектуру веб-серверов IIS и Apache, а также FTP-серверов на платформах Windows и Linux.
- **Практическая:** Приобрести навыки установки, настройки и администрирования HTTP- и FTP-серверов в Windows Server 2008 (IIS) и Debian (Apache, vsftpd), настройки брандмауэра, управления виртуальными каталогами и проверки работоспособности служб.

## **Задачи**
1. Установить и настроить веб-сервер IIS с поддержкой FTP в Windows Server 2008.
2. Настроить правила брандмауэра для доступа к HTTP и FTP.
3. Создать виртуальный каталог и проверить доступ к веб-сайту.
4. Установить и настроить веб-сервер Apache в Debian.
5. Изменить стартовую страницу и проверить доступ к серверу.
6. Освоить управление службой Apache с помощью утилиты apache2ctl.
7. Выполнить HTTP-запрос вручную через telnet.
8. Установить и настроить FTP-сервер vsftpd в Debian.
9. Проверить работу FTP-сервера и проанализировать подключения.

## **Оборудование и программное обеспечение**

- **Оборудование (виртуальное):**
  - Виртуальная машина №1: Windows Server 2008 R2 (2 ГБ ОЗУ, 2 ЦП, 30 ГБ HDD).

  - Сетевой режим: мостовой адаптер или внутренняя сеть для взаимодействия ВМ.

## **Краткие теоретические сведения**

**HTTP (HyperText Transfer Protocol)** — протокол прикладного уровня, используемый для передачи гипертекстовых документов (веб-страниц). Веб-сервер обрабатывает HTTP-запросы и возвращает соответствующие ресурсы.

**IIS (Internet Information Services)** — набор служб для развертывания веб- и FTP-серверов в Windows. Поддерживает ASP.NET, PHP и другие технологии.

**Apache HTTP Server** — кроссплатформенный веб-сервер с открытым исходным кодом. Один из самых популярных серверов в интернете.

**FTP (File Transfer Protocol)** — протокол передачи файлов между клиентом и сервером. Может работать в активном и пассивном режимах.

**vsftpd (Very Secure FTP Daemon)** — FTP-сервер для Unix-подобных систем, ориентированный на безопасность и производительность.

## **Порядок выполнения работы**

### **1. Подготовительный этап**
1. Запустите Oracle VM VirtualBox.
2. Подготовьте две виртуальные машины: одну с Windows Server 2008 R2, другую с Debian.
3. Убедитесь, что обе машины имеют сетевые подключения и могут взаимодействовать друг с другом (например, через мостовой адаптер).
4. Войдите в каждую систему с правами администратора/root.

### **2. Основной этап**

#### **Часть 1. Настройка HTTP и FTP в Windows Server 2008**

**1. Установка роли веб-сервера (IIS) с поддержкой FTP**
   - Откройте **Диспетчер сервера** → **Роли** → **Добавить роли**.
   - Выберите **Веб-сервер (IIS)**.
   - В мастере установки на шаге выбора служб роли убедитесь, что выбраны:
     - **Службы FTP** (FTP Server, FTP Management Console).
   - Завершите установку.

**2. Настройка брандмауэра**
   - Откройте **Панель управления** → **Брандмауэр Windows** → **Разрешение запуска программы или компонента через брандмауэр Windows**.
   - Установите флажки для **FTP Server** и **World Wide Web Services (HTTP)**.
   - Нажмите **ОК**.

**3. Создание виртуального каталога для веб-сайта**
   - Откройте **Диспетчер служб IIS** (Пуск → Администрирование).
   - В дереве подключений выберите **Сайты** → **Default Web Site**.
   - В области действий выберите **Добавить виртуальный каталог**.
   - Укажите:
     - Псевдоним: **mypage**
     - Физический путь: **C:\mysite** (предварительно создайте папку).
   - Нажмите **ОК**.

**4. Настройка FTP-сервера**
   - В диспетчере IIS выберите узел сервера, затем дважды щелкните **FTP-авторизация**.
   - Отключите анонимный доступ, оставив только **Базовую проверку подлинности**.
   - Настройте правила авторизации, разрешив доступ указанным пользователям.
   - В оснастке **Службы** (Пуск → Администрирование) найдите службу **FTP Publishing Service**.
   - Установите тип запуска **Автоматически** и запустите службу.

**5. Проверка работы серверов**
   - Создайте в каталоге `C:\mysite` файл `index.html` с простым html контентом.
   - Откройте браузер на локальной машине Windows и перейдите по адресу:
     - `http://localhost/` — должна открыться ваша страница `index.html`.
     - `http://localhost/mypage` — вы должны увидеть содержимое папки C:\mysite (если там есть файлы, например, index.html).
   - В браузере введите `ftp://localhost` — должно появиться окно входа с запросом учетных данных.

**6. Просмотр текущих подключений к FTP**
   - В диспетчере IIS выберите **FTP-сервер** → **Текущие сеансы**, чтобы увидеть подключенных пользователей.

#### **Часть 2. Настройка HTTP и FTP в Debian**

**1. Установка веб-сервера Apache**
   - Откройте терминал.
   - Выполните обновление списка пакетов:
     ```bash
     sudo apt update
     ```
   - Установите Apache2:
     ```bash
     sudo apt install apache2 -y
     ```

**2. Проверка работоспособности Apache**
   - Откройте браузер и перейдите по адресу `http://127.0.0.1` или `http://<IP-адрес_вашей_ВМ>`.
   - Должна отобразиться стандартная страница Apache2 Ubuntu Default Page.

**3. Изменение стартовой страницы**
   - Файлы сайта находятся в `/var/www/html/`.
   - Отредактируйте `index.html`
   - Добавьте в тело документа свою фамилию, например: `<p>Ivan Ivanov</p>`.
   - Обновите страницу в браузере — ваша фамилия должна отображаться.

**4. Просмотр сервера соседа**
   - Узнайте IP-адрес соседней ВМ Debian (команда `ip a`).
   - Откройте браузер и введите `http://<IP_соседа>` — вы должны увидеть его страницу с его фамилией.

**5. Управление службой Apache через systemctl**
   - Остановите сервер:
     ```bash
     systemctl stop apache2
     ```
   - Проверьте, что страница перестала открываться.
   - Запустите сервер:
     ```bash
	 systemctl start apache2
     ```
   - Перезапустите:
     ```bash
	 systemctl restart apache2
     ```

**6. Отключение автозапуска Apache (опционально)**
   - Выполните команду:
     ```bash
	 systemctl disable apache2
     ```
   - Для возврата автозапуска: `systemctl enable apache2`.

**7. Выполнение HTTP-запроса вручную через telnet**
   - Установите telnet, если он не установлен:
     ```bash
     sudo apt install telnet -y
     ```
   - Выполните:
     ```bash
     telnet 127.0.0.1 80
     ```
   - После подключения введите:
     ```
     GET /index.html HTTP/1.1
     Host: 127.0.0.1
     (пустая строка)
     (пустая строка)
     ```
   - Вы увидите ответ сервера: заголовки и содержимое страницы.
   - Проанализируйте структуру HTTP-ответа.

**8. Установка FTP-сервера vsftpd**
   - Выполните:
     ```bash
     sudo apt install vsftpd -y
     ```
   - По умолчанию создается пользователь `ftp` с домашним каталогом `/srv/ftp`.
   - Настройте конфигурацию (при необходимости):
     ```bash
     sudo nano /etc/vsftpd.conf
     ```
     - Разрешите локальным пользователям вход: `local_enable=YES`
     - Разрешите запись: `write_enable=YES`
   - Перезапустите службу:
     ```bash
     sudo systemctl restart vsftpd
     ```

**9. Проверка FTP-сервера**
   - Откройте браузер и введите `ftp://127.0.0.1`. Если настроена анонимная авторизация, вы увидите содержимое `/srv/ftp`. Иначе потребуется ввести учетные данные.
