## 1. Тема
Изучение протокола SNMP (Simple Network Management Protocol) и настройка мониторинга сетевых устройств с использованием утилит Linux и симулятора Cisco Packet Tracer.

## 2. Цель работы
- **Теоретическая:** Изучить архитектуру протокола SNMP, его компоненты (менеджер, агент, MIB), версии и основные операции.
- **Практическая:** Приобрести навыки работы с утилитами командной строки SNMP в ОС Linux Manjaro, а также настройки SNMP-агента на маршрутизаторе Cisco в среде Packet Tracer и управления им с помощью MIB-браузера.

## 3. Задачи
1. Установить пакеты для работы с SNMP в Manjaro.
2. Выполнить базовые запросы к SNMP-агенту с помощью утилит `snmptranslate`, `snmpget`, `snmpgetnext`, `snmpwalk`, `snmptable`.
3. Проанализировать структуру MIB и значения системных переменных удалённого агента (IP 10.4.9.100, community `demopublic`).
4. В Cisco Packet Tracer построить простейшую сеть из маршрутизатора и ПК.
5. Настроить SNMP-агент на маршрутизаторе (строки community read-only и read-write).
6. С помощью MIB-браузера на ПК выполнить запросы к маршрутизатору: получить системную информацию, изменить имя устройства, управлять состоянием интерфейса.
7. Задокументировать все этапы с помощью скриншотов.

## 4. Оборудование и программное обеспечение
- **Программное обеспечение:**
  - Операционная система Manjaro Linux (установленная на физическом компьютере или в виртуальной машине).
  - Пакет `net-snmp` (утилиты командной строки).
  - Cisco Packet Tracer (любая современная версия, например 8.x).

## 5. Краткие теоретические сведения

**SNMP (Simple Network Management Protocol)** – протокол прикладного уровня, предназначенный для управления и мониторинга сетевых устройств (маршрутизаторов, коммутаторов, серверов, принтеров и т.д.). Он позволяет централизованно собирать информацию о состоянии устройств, изменять их конфигурацию и получать уведомления о событиях.

**Основные компоненты SNMP:**
- **Управляющая система (Manager)** – приложение, которое отправляет запросы агентам и обрабатывает полученные данные.
- **Агент (Agent)** – программный модуль, работающий на управляемом устройстве. Он хранит информацию в виде переменных и отвечает на запросы менеджера.
- **База управляющей информации (MIB, Management Information Base)** – иерархическая база данных, описывающая структуру управляемых объектов. Каждый объект имеет уникальный идентификатор – **OID (Object Identifier)**.

**Версии SNMP:**
- **SNMPv1** – первая версия, использует простую аутентификацию на основе community-строк.
- **SNMPv2c** – расширенная версия с новыми типами PDU (GetBulk), также использует community-строки.
- **SNMPv3** – обеспечивает шифрование и аутентификацию, повышая безопасность.

**Основные операции SNMP:**
- `GET` – запрос значения конкретной переменной.
- `GETNEXT` – запрос следующей переменной в иерархии MIB (используется для обхода таблиц).
- `GETBULK` – массовое получение больших объёмов данных (только в SNMPv2c и v3).
- `SET` – изменение значения переменной (конфигурирование устройства).
- `TRAP` / `INFORM` – асинхронные уведомления от агента менеджеру о событиях.

**Утилиты командной строки net-snmp:**
- `snmptranslate` – преобразование OID из числового формата в текстовый и обратно.
- `snmpget` – выполнение GET-запроса.
- `snmpgetnext` – выполнение GETNEXT-запроса.
- `snmpwalk` – многократный вызов GETNEXT для получения целой ветки MIB.
- `snmptable` – вывод содержимого таблицы MIB в удобном табличном виде.

## 6. Порядок выполнения работы

### 6.1. Подготовительный этап
1. Убедитесь, что Manjaro Linux установлен и имеет доступ в сеть (проверьте связь с агентом 10.4.9.100 командой `ping 10.4.9.100`).
2. Установите пакет `net-snmp` и, при необходимости, дополнительные MIB-файлы:
   ```bash
   sudo pacman -S net-snmp
   ```
3. Для корректного отображения текстовых имён MIB отредактируйте файл `/etc/snmp/snmp.conf`:
   ```bash
   sudo nano /etc/snmp/snmp.conf
   ```
   Закомментируйте строку, начинающуюся с `mibs` (если она есть), добавив символ `#` в начало:
   ```
   #mibs : ALL
   ```
   Сохраните файл. Это позволит утилитам загружать все доступные MIB-модули.

### 6.2. Основной этап

#### Часть 1. Работа с утилитами SNMP в Manjaro (агент 10.4.9.100)

**1. Изучение утилиты `snmptranslate`**
   - Выполните преобразование числового OID `.1.3.6.1.2.1.1.3.0` в текстовое имя:
     ```bash
     snmptranslate .1.3.6.1.2.1.1.3.0
     ```
   - Выполните обратное преобразование для OID `1.3.6.1.2.1.5.1`:
     ```bash
     snmptranslate 1.3.6.1.2.1.5.1
     ```
   - Запишите результаты. Утилита показывает полное имя объекта в дереве MIB.

**2. Запрос одной переменной – `snmpget`**
   - Получите значение системного времени работы агента (sysUpTime.0):
     ```bash
     snmpget -v 2c -c demopublic 10.4.9.100 system.sysUpTime.0
     ```
   - Обратите внимание на формат ответа: значение в сотых долях секунды (timeticks).

**3. Запрос следующей переменной – `snmpgetnext`**
   - Выполните запрос, который вернёт следующую переменную после sysUpTime.0:
     ```bash
     snmpgetnext -v 2c -c demopublic 10.4.9.100 system.sysUpTime.0
     ```
   - Сравните результат с предыдущим.

**4. Обход ветки MIB – `snmpwalk`**
   - Получите все переменные ветки `system` (или всей MIB-2, если не указан OID):
     ```bash
     snmpwalk -v 2c -c demopublic 10.4.9.100 system
     ```
   - Изучите выведенный список: sysDescr, sysObjectID, sysUpTime, sysContact, sysName, sysLocation, sysServices.
   - Запишите значения для своей отчётности.

**5. Получение таблицы – `snmptable`**
   - Выведите таблицу системных объектов (sysORTable), содержащую информацию о поддерживаемых модулях:
     ```bash
     snmptable -v 2c -c demopublic 10.4.9.100 sysORTable
     ```
   - Проанализируйте структуру таблицы и её содержимое.

#### Часть 2. Настройка SNMP в Cisco Packet Tracer

**1. Построение топологии**
   - Откройте Cisco Packet Tracer.
   - Добавьте на рабочую область маршрутизатор Router-PT и персональный компьютер PC-PT.
   - Соедините их прямым медным кабелем (через порт FastEthernet0/0 маршрутизатора и FastEthernet0 ПК).

**2. Настройка IP-адресации**
   - На маршрутизаторе:
     ```cisco
     Router> enable
     Router# configure terminal
     Router(config)# interface fastEthernet 0/0
     Router(config-if)# ip address 192.168.1.1 255.255.255.0
     Router(config-if)# no shutdown
     Router(config-if)# exit
     Router(config)# exit
     Router# write
     ```
   - На ПК: откройте вкладку Desktop → IP Configuration. Задайте:
     - IP-адрес: 192.168.1.100
     - Маска: 255.255.255.0
     - Шлюз: 192.168.1.1

**3. Настройка SNMP на маршрутизаторе**
   - В режиме глобальной конфигурации выполните:
     ```cisco
     Router(config)# snmp-server community public ro
     Router(config)# snmp-server community private rw
     Router(config)# snmp-server location "Lab"
     Router(config)# snmp-server contact "student@lab.local"
     Router(config)# exit
     Router# write
     ```
     Здесь `public` – строка read-only (только для чтения), `private` – read-write (чтение и запись). Запомните их.

**4. Настройка MIB-браузера на ПК**
   - На ПК перейдите в Desktop → MIB Browser.
   - В поле **Address** укажите IP-адрес маршрутизатора: **192.168.1.1**.
   - **Port** оставьте 161.
   - **Read Community** введите: `public`.
   - **Write Community** введите: `private`.
   - **SNMP Version** выберите v1 или v2c (для простоты v1).

**5. Получение информации с маршрутизатора**
   - В выпадающем списке OID выберите `system` (или вручную введите OID 1.3.6.1.2.1.1).
   - Нажмите кнопку **Go** (или Get). Вы увидите значения системных переменных (sysName, sysDescr и др.).
   - Запишите полученные данные.

**6. Изменение имени устройства (SET-операция)**
   - В поле OID выберите `sysName.0` (или вручную 1.3.6.1.2.1.1.5.0).
   - В поле **Data Type** выберите `Octet String`.
   - В поле **Value** введите новое имя, например `MyRouter`.
   - Нажмите кнопку **Set**. Если операция выполнена успешно, появится подтверждение.
   - Снова выполните Get для sysName.0, чтобы убедиться, что имя изменилось.

**7. Управление интерфейсом**
   - Найдите OID для состояния интерфейса `ifAdminStatus` (ветка 1.3.6.1.2.1.2.2.1.7). Для интерфейса FastEthernet0/0 это может быть, например, .1.3.6.1.2.1.2.2.1.7.1 (где последняя цифра – индекс интерфейса).
   - С помощью Get убедитесь, что текущее состояние – 1 (up).
   - Выполните Set со значением 2 (down), чтобы отключить интерфейс.
   - Проверьте, что интерфейс действительно отключился (например, через CLI маршрутизатора или повторный Get).
