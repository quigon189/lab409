## 1. Тема
Учет трафика, мониторинг компьютерных сетей и анализ сети с помощью команд сетевой операционной системы.

## 2. Цель работы
- **Теоретическая:** Изучить методы учета сетевого трафика, архитектуру и принципы работы современных систем мониторинга (Prometheus, Grafana, ntopng), а также освоить средства анализа сети с использованием встроенных команд ОС и специализированных утилит.
- **Практическая:** Приобрести навыки развертывания систем мониторинга в Docker-контейнерах, настройки сбора метрик с реального оборудования (хостовой ПК, маршрутизатора/коммутатора Cisco), анализа сетевого трафика с помощью tcpdump, а также использования команд сетевой диагностики (ping, traceroute, netstat, ss, ip, arp, route, nmap, iperf3).

## 5. Краткие теоретические сведения

### 5.1. Учет трафика и мониторинг сети

**Учет трафика (Traffic Accounting)** — процесс сбора, анализа и хранения информации о проходящем через сеть трафике. Основные протоколы учета трафика:

- **NetFlow (Cisco)** — экспорт агрегированных потоков данных с маршрутизаторов и коммутаторов (5-7 полей: src/dst IP, порты, протокол). Позволяет анализировать, кто с кем и сколько обменивается данными.
- **sFlow (Sampled Flow)** — статистический (выборочный) метод, берет выборку пакетов, не строит потоков. Идеален для высокоскоростных линков и дает видимость VLAN/MAC.
- **IPFIX (IP Flow Information Export)** — стандартизированная версия NetFlow v9 (IETF), более гибкий и расширяемый шаблонный протокол.

**Современные системы мониторинга сети**:

| Система        | Назначение                                                                      | Особенности                                                                           |
| -------------- | ------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------- |
| **Zabbix**     | Мониторинг доступности, производительности, сбор метрик (SNMP, агентский опрос) | Мощная система оповещения, веб-интерфейс, поддержка шаблонов                          |
| **Prometheus** | Сбор временных рядов метрик (pull-модель)                                       | Интеграция с Grafana, поддержка экспортеров (SNMP, Node), гибкий язык запросов PromQL |
| **Grafana**    | Визуализация данных (интерактивные дашборды)                                    | Поддержка множества источников данных: Prometheus, Zabbix, Elasticsearch              |
| **ntopng**     | Анализ трафика в реальном времени                                               | Показывает хосты, протоколы, приложения, поддерживает NetFlow/sFlow/IPFIX             |
| **LibreNMS**   | Автообнаружение устройств, опрос через SNMP                                     | Автоматическое построение карт сети                                                   |

### 5.2. Средства анализа сети

**Команды сетевой ОС (Linux):**

| Команда                                        | Назначение                                            |
| ---------------------------------------------- | ----------------------------------------------------- |
| `ping`                                         | Проверка доступности узла, измерение RTT              |
| `traceroute / tracepath`                       | Определение маршрута следования пакетов               |
| `netstat -tulpn` / `ss -tulpn`                 | Просмотр открытых портов и прослушиваемых сокетов     |
| `ip addr` / `ip route`                         | Просмотр IP-адресов и таблицы маршрутизации           |
| `arp -a` / `ip neigh`                          | Просмотр ARP-таблицы                                  |
| `tcpdump -i eth0`                              | Захват пакетов в реальном времени                     |
| `nmap -sS 192.168.1.0/24`                      | Сканирование сети, обнаружение хостов и сервисов      |
| `iperf3 -s` (сервер) / `iperf3 -c <server_ip>` | Тестирование пропускной способности                   |
| `snmpwalk -v2c -c public <ip> system`          | Получение полного поддерева MIB от SNMP-агента        |
| `wireshark`                                    | Графический анализатор трафика (или `tshark` для CLI) |

**Основные OID для мониторинга сети (MIB-2, префикс 1.3.6.1.2.1.1):**

- `.1.3.6.1.2.1.1.1.0` (sysDescr) — текстовое описание системы.
- `.1.3.6.1.2.1.1.3.0` (sysUpTime) — время работы системы (timeticks).
- `.1.3.6.1.2.1.1.5.0` (sysName) — имя системы (доменное имя).
- `.1.3.6.1.2.1.1.6.0` (sysLocation) — физическое расположение.
- `.1.3.6.1.2.1.2.2.1.2` (ifDescr) — описание интерфейсов.
- `.1.3.6.1.2.1.2.2.1.10` (ifInOctets) — входящие октеты.
- `.1.3.6.1.2.1.2.2.1.16` (ifOutOctets) — исходящие октеты.

### 5.3. Протокол SNMP

**SNMP (Simple Network Management Protocol)** — протокол прикладного уровня для управления и мониторинга сетевых устройств. Использует UDP-порты 161 (запросы) и 162 (trap-сообщения). Агент хранит управляющую информацию в виде переменных, организованных в иерархическое дерево MIB. Каждая переменная идентифицируется OID.

**Типы PDU (SNMPv2c):**

| Тип PDU        | Код | Назначение                             |
| -------------- | --- | -------------------------------------- |
| GetRequest     | 0   | Запрос значения переменной             |
| GetNextRequest | 1   | Запрос следующей переменной в иерархии |
| GetResponse    | 2   | Ответ на запрос                        |
| SetRequest     | 3   | Установка значения переменной          |
| GetBulkRequest | 5   | Массовое получение данных              |

### 6.2. Основной этап

#### 1. Базовые средства анализа сети

**1. Проверка связности и маршрутизация**
   - Выполните `ping` до шлюза по умолчанию и до удаленного узла (например, 8.8.8.8).
   - Выполните `traceroute` до lab409.cloudpub.ru и проанализируйте промежуточные узлы.
   - Просмотрите таблицу маршрутизации: `ip route show` или `route -n`.
   - Просмотрите ARP-таблицу: `ip neigh show` или `arp -a`.

**2. Просмотр сетевых интерфейсов и портов**
   - Просмотрите список сетевых интерфейсов: `ip addr show` или `ifconfig -a`.
   - Просмотрите список открытых портов: `ss -tulpn` или `netstat -tulpn`.
   - Определите, какие службы прослушивают порты 80, 443, 22, 53.

**3. Работа с tcpdump**
   - Определите имя сетевого интерфейса: `ip link show`.
   - Выполните захват ICMP-пакетов на 10 секунд:
     ```bash
     sudo tcpdump -i <interface> icmp -c 20 -v
     ```
   - Выполните ping до другого узла и проанализируйте захваченные пакеты (структуру ICMP Echo Request/Reply, MAC-адреса).

#### 2. Развертывание систем мониторинга в Docker

**1. Развертывание стека Prometheus + Grafana + SNMP Exporter**
   - Создайте директорию для проекта:
     ```bash
     mkdir -p ~/monitoring/prometheus
     mkdir -p ~/monitoring/grafana
     cd ~/monitoring
     ```
   - Создайте файл `docker-compose.yml`:
     ```yaml
     services:
       prometheus:
         image: prom/prometheus:latest
         container_name: prometheus
         volumes:
           - ./prometheus/prometheus.yml:/etc/prometheus/prometheus.yml
         ports:
           - "9090:9090"
         restart: unless-stopped
     
       grafana:
         image: grafana/grafana:latest
         container_name: grafana
         ports:
           - "3000:3000"
         environment:
           - GF_SECURITY_ADMIN_PASSWORD=admin
         volumes:
           - ./grafana:/var/lib/grafana
         restart: unless-stopped
     
       snmp-exporter:
         image: prom/snmp-exporter:latest
         container_name: snmp-exporter
         ports:
           - "9116:9116"
         restart: unless-stopped
     ```
   - Создайте конфигурационный файл `prometheus/prometheus.yml`:
     ```yaml
     global:
       scrape_interval: 15s
     
     scrape_configs:
       - job_name: 'snmp'
         static_configs:
           - targets:
             - '10.4.9.100'  # IP SNMP-агента
         metrics_path: /snmp
         params:
           module: [if_mib]
         relabel_configs:
           - source_labels: [__address__]
             target_label: __param_target
           - source_labels: [__param_target]
             target_label: instance
           - target_label: __address__
             replacement: snmp-exporter:9116
     ```
   - Запустите стек:
     ```bash
     docker compose up -d
     ```
   - Проверьте работу: откройте в браузере Grafana (`http://localhost:3000`, логин `admin`, пароль `admin`). Добавьте источник данных Prometheus (`http://prometheus:9090`). Импортируйте готовый дашборд для SNMP (ID 11175).
   - После проведения работы остановите контейнеры:
   ```bash
   docker compose down
   ```

**2. Развертывание ntopng для анализа трафика**
   - Создайте директорию `~/ntopng` и файл `docker-compose.yml`:
     ```yaml
     services:
       ntopng:
         image: ntop/ntopng:latest
         container_name: ntopng
         network_mode: host
         environment:
           - NTOPNG_INTERFACE=eth0   # замените на ваш интерфейс
           - NTOPNG_PORT=3001
         volumes:
           - ./data:/var/lib/ntopng
         restart: unless-stopped
     ```
   - Запустите ntopng:
     ```bash
     docker compose up -d
     ```
   - Откройте в браузере `http://localhost:3001`. Изучите интерфейс: вкладки Hosts, Protocols, Flows.

#### 3. Мониторинг реального оборудования (Cisco)

Данная часть выполняется в аудитории 409 на оборудовании на стойке. Уточните у преподавателя адреса и учетные данные.

**1. Настройка SNMP на маршрутизаторе Cisco (реальном или в Packet Tracer)**
   ```cisco
   Router> enable
   Router# configure terminal
   Router(config)# snmp-server community public ro
   Router(config)# snmp-server community private rw
   Router(config)# snmp-server location "Laboratory"
   Router(config)# snmp-server contact "student@lab.local"
   Router(config)# snmp-server host 192.168.1.100 public
   Router(config)# snmp-server enable traps
   Router(config)# exit
   Router# write
   ```

**2. Настройка NetFlow/sFlow на Cisco (для учета трафика)**
   - Настройка NetFlow (пример для FastEthernet 0/0):
     ```cisco
     Router(config)# interface fastEthernet 0/0
     Router(config-if)# ip route-cache flow
     Router(config-if)# exit
     Router(config)# ip flow-export version 9
     Router(config)# ip flow-export destination 192.168.1.100 2055
     Router(config)# ip flow-export source fastEthernet 0/0
     ```
   - После настройки данные NetFlow будут отправляться на коллектор (например, ntopng или ELK).

**3. Опрос SNMP-агента Cisco с помощью snmpwalk**
   ```bash
   # Получение системной информации
   snmpwalk -v 2c -c public 192.168.1.1 system
   
   # Получение информации об интерфейсах
   snmpwalk -v 2c -c public 192.168.1.1 interfaces
   
   # Получение загрузки CPU
   snmpwalk -v 2c -c public 192.168.1.1 .1.3.6.1.4.1.9.2.1.58
   ```

**4. Добавление устройства Cisco в Prometheus (через snmp-exporter)**
   - Добавьте в `prometheus/prometheus.yml` новый target:
     ```yaml
     - targets:
       - '192.168.1.1'   # IP маршрутизатора
     ```
   - Перезапустите Prometheus: `docker compose restart prometheus`.

#### 4. Расширенный анализ сети

**1. Сканирование сети с помощью nmap**
Предоставлен пример для подсети 192.168.1.0/24, замените ее на вашу подсеть.
   ```bash
   # Обнаружение хостов в локальной сети
   nmap -sn 192.168.1.0/24
   
   # Сканирование открытых портов
   nmap -sS 192.168.1.1
   
   # Определение ОС и версий сервисов
   sudo nmap -O -sV 192.168.1.1
   
   # Сканирование UDP-портов
   sudo nmap -sU 192.168.1.1
   ```

**2. Тестирование пропускной способности с iperf3**
   - На одном ПК запустите сервер:
     ```bash
     iperf3 -s
     ```
   - На другом ПК (или в той же сети) запустите клиент:
     ```bash
     iperf3 -c <IP_сервера> -t 10
     ```
   - Проанализируйте результаты: скорость передачи (Bitrate), количество переданных данных.
