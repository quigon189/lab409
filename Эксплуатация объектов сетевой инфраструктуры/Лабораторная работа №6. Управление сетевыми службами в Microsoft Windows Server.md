## **Тема**
Установка, настройка и администрирование сетевых служб DNS и DHCP в Microsoft Windows Server.

## **Цель работы**
- **Теоретическая:** Изучить принципы работы и назначение сетевых служб DNS и DHCP в корпоративных сетях.
- **Практическая:** Приобрести навыки развертывания, настройки и диагностики служб DNS и DHCP на Windows Server 2008, а также работы с межсетевым экраном и средствами мониторинга.

## **Задачи**
1. Настроить сетевые параметры виртуальной машины Windows Server.
2. Установить и настроить роль DNS-сервера, создать прямую и обратную зоны.
3. Настроить правила брандмауэра для работы DNS-сервера.
4. Создать и проверить различные типы DNS-записей (A, CNAME, поддомены).
5. Установить и настроить роль DHCP-сервера, создать область адресов.
6. Изучить настройки брандмауэра для DHCP-сервера.
7. Проверить работоспособность служб с помощью диагностических утилит.

## **Оборудование и программное обеспечение**
- **Программное обеспечение:** Oracle VM VirtualBox, Windows Server.
- **Оборудование (виртуальное):**
  - Виртуальная машина: Windows Server.
  - Сетевой адаптер: мостовой режим (Bridged Adapter), неразборчивый режим — разрешить все.

## **Краткие теоретические сведения**

**DNS (Domain Name System)** — иерархическая распределенная система преобразования доменных имен в IP-адреса и наоборот. Состоит из серверов, хранящих зоны и ресурсные записи (A, CNAME, MX, PTR и др.).

**DHCP (Dynamic Host Configuration Protocol)** — протокол автоматической настройки сетевых параметров клиентских устройств (IP-адрес, маска, шлюз, DNS-сервер, время аренды).

**Зона прямого просмотра** содержит сопоставления доменных имен IP-адресам.

**Зона обратного просмотра** содержит сопоставления IP-адресов доменным именам.

**nslookup** — сетевая утилита командной строки для запросов к DNS-серверам и диагностики работы DNS.

## **Порядок выполнения работы**

### **1. Подготовительный этап**
1. Запустите Oracle VM VirtualBox.
2. Запустите виртуальную машину с установленной ОС Windows Server.
3. Войдите в систему с правами администратора.

### **2. Основной этап**

#### **Часть 1. Настройка DNS-сервера**

**1. Настройка сети**
   - В VirtualBox настройте сетевой адаптер в режиме "Мостовой адаптер", разрешите неразборчивый режим.
   - В Windows Server назначьте статический IP-адрес из диапазона сети лаборатории 409 (192.168.0.0/16).
   - Проверьте связь с шлюзом и DNS-сервером колледжа:
     ```cmd
     ping <IP_шлюза>
     ping <IP_DNS_сервера>
     ```

**2. Установка роли DNS-сервера**
   - Откройте **Диспетчер сервера** → **Роли** → **Добавить роли**.
   - Выберите **DNS-сервер** и завершите установку.

**3. Изучение службы DNS**
   - Откройте **Администрирование** → **Службы**.
   - Найдите службу **DNS-сервер**. Запишите:
     - Описание
     - Состояние
     - Тип запуска
     - Учетная запись для входа
   - Откройте **Свойства** → вкладка **Восстановление**, запишите действия при сбое службы.

**4. Настройка брандмауэра для DNS**
   - Откройте **Брандмауэр Windows в режиме повышенной безопасности**.
   - Найдите все правила для входящих и исходящих подключений, связанные с DNS-сервером.
   - Для каждого правила запишите:
     - Имя и описание
     - Тип протокола и номер
     - Локальный и удаленный порты

**5. Настройка пересылки DNS**
   - Откройте **DNS-менеджер**.
   - ПКМ на имени сервера → **Свойства** → **Сервер пересылки**.
   - Добавьте IP-адрес DNS-сервера учебного заведения (10.7.2.1).

**6. Создание зоны прямого просмотра**
   - Создайте **Основную зону** с именем: **ваша-фамилия.local** (например, **ivanov.local**).
   - Запретите динамические обновления.

**7. Проверка разрешения имени сервера**
   - С основной ОС выполните:
     ```cmd
     nslookup имя-сервера.ваша-фамилия.local <IP_DNS_сервера>
     ```
   - Пример: `nslookup 409-ivanov.ivanov.local 192.168.1.10`

**8. Создание записей типа A**
   - Добавьте записи для:
     - `file-server.ваша-фамилия.local`
     - `app-server.ваша-фамилия.local`
     - `print-server.ваша-фамилия.local`
   - IP-адреса выберите из той же подсети, что и DNS-сервер.

**9. Проверка разрешения имен**
   - Используйте nslookup для проверки всех созданных записей A.

**10. Создание зоны обратного просмотра**
    - Создайте **Основную зону обратного просмотра IPv4** для своей подсети.

**11. Добавление PTR-записей**
    - В обратную зону добавьте записи для DNS-сервера, file-server, app-server, print-server.

**12. Проверка обратного разрешения**
    - Используйте nslookup для проверки разрешения IP-адресов в имена.

**13. Создание записей CNAME**
    - Создайте псевдонимы:
      - `apps` для `app-server`
      - `files` для `file-server`
      - `print` для `print-server`
    - Проверьте через nslookup.

**14. Создание поддомена**
    - В зоне создайте поддомен `hr.ваша-фамилия.local`.
    - Добавьте запись A для `pc1` с выбранным IP-адресом.
    - Проверьте разрешение имени.

**15. Тестирование приостановленной службы**
    - Приостановите службу DNS-сервер.
    - Попробуйте разрешить имена через nslookup.
    - Запишите результат.
    - Запустите службу снова.

#### **Часть 2. Настройка DHCP-сервера**

**1. Установка роли DHCP-сервера**
   - Откройте **Диспетчер сервера** → **Добавить роли**.
   - Выберите **DHCP-сервер**.
   - Настройте:
     - Привязку к статическому IP-адресу сервера
     - Родительский домен: **ваша-фамилия.local**
     - WINS не требуется
     - Создайте область: **ваша-фамилияScope** с диапазоном IP-адресов
     - Укажите шлюз по умолчанию
     - Отключите DHCPv6

**2. Изучение службы DHCP**
   - В оснастке **Службы** найдите **DHCP-сервер**.
   - Запишите те же параметры, что и для DNS.
   - Изучите вкладку **Восстановление**.

**3. Настройка брандмауэра для DHCP**
   - Аналогично DNS найдите и запишите все правила брандмауэра для DHCP.

**4. Изучение консоли DHCP**
   - Откройте **DHCP-менеджер**.
   - Изучите структуру: области, резервирования, аренды адресов.

**5. Тестирование DHCP**
   - Настройте другое устройство на получение IP-адреса автоматически.
   - Убедитесь, что оно получает адрес из созданной области.

### **3. Контрольный этап**

2. Проверьте настройки брандмауэра для обеих служб.
